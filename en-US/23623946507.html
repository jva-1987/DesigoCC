Export Recipients

This section describes additional procedures exporting recipients.

Export Recipients to XML

The Export feature is disabled for Windows App client.

  1. Select Application > Notification > Recipients.
  2. The Recipients Editor tab displays.
  3. Select the Export tab.
  4. Open the Export to XML expander.
  5. Select the recipients to be exported and click to move the selection to the Selected for export text box.
    NOTE: Search recipients to narrow down the list by typing in the recipient name.
  6. Click Export.
  7. The Save dialog box displays.
  8. Select the location where the XML file must be saved.
  9. Enter a file name.
  10. Click Save.
    NOTE: XML format is meant for system to system export of recipients.
  1. The recipients are exported in XML format.

Export Recipients to XML using Command

  1. Select Applications > Notification > Recipients.
  2. Click the Export XML link.
  3. The Recipients Editor tab displays.
  4. Select the Extended Operation tab.
  5. Click the Export XML link to enable the export options.
  6. The File Path field and Send display.
  7. In the File Path field, enter the local file path of the MNS server where the XML file must be saved.
  8. Click Send.
  1. All the recipients in the Notification are exported to the XML file specified in the file path.

Export Recipients to XML using Reaction

  1. Select Applications > Logics > Reactions.
  2. The Reaction Editor tab displays.
  3. Drag the recipients node from Applications > Notification > Recipients onto the Action expander of the Output expander.
  4. Select Export XML[Commands.XMLExport.Request.Export] from the Property drop-down list.
  5. Select ExportXML[ExportXML] command to execute.
  6. In the ServerFilePath field, enter the local file path of the MNS server where the XML file must be saved.
  7. Open the Triggers expander. From the drop-down list, select the appropriate logical operator (AND, OR).
  8. Open the Time and Organization Mode expander.
  9. Click Add to add a new time row and enter a new time or schedule that periodically triggers the Export XML command.
  10. Click Save As and enter the name and description for the Export XML reaction.
  1. The new Export XML reaction displays in the System Browser.

Configure Mappings for CSV Export

Before configuring the CSV mapping, the user first needs to determine three properties of the CSV files:

  • Header row: The first row in a CSV file may contain column names. If a header row is present, columns can be referenced by name in the CSV mapping. Otherwise, columns will need to be referenced by their index (position) in the CSV file.
  • Delimiter: A certain character that delimits two values in a CSV file. Notification supports commas, semicolons, spaces and tabs as delimiter.
  • Quotation mark: Surrounding individual values with quotation marks allows the use of delimiter characters within values. Notification supports single quotes and double quotes as quotation marks.
    NOTE 1: CSV files exported from Excel by default have the following properties:
    Header row: Yes
    Delimiter: Comma
    Quotation mark: Double quote.
    NOTE 2: If the character used as quotation mark appears within quoted values, the character must be escaped by doubling it. This escaping process is typically performed by the process that creates a CSV file from a data source.
  1. Do the following to create a new export mapping:
    a. Click New: The Create mapping as dialog box displays.
    b. Enter a name for the mapping and click OK: The name displays under Mapping.
    c. (Optional) Select the Header row check box if the exported CSV file must contain a header row.
    d. Select a separator from the Delimiter drop-down list.
    e. In the Columns section, map a Notification Property with a CSV Column. From the Notification Property drop-down list, select a Notification Property and enter the information in the CSV Column (Excel Column) field for the selected notification property.
    f. From the Quotations drop-down list, select a quotation type.
    g. Click Save.
  2. The mapping configuration is saved.
    NOTE: This configured mapping can be saved for future use.
  3. Do the following to edit an existing mapping:
    a. Select the mapping from the Mapping drop-down list.
    b. Edit the required fields.
    c. Click Save.
  4. To select an existing mapping, select it from the Mapping drop-down list.
  5. Do the following to save an existing mapping under a new name:
    a.     Select the mapping from the Mapping drop-down list.
    b. Click Save As.
    c. The Save mapping as dialog box displays.
    NOTE: By default, Copy of [Mapping Name] displays in the Save mapping as dialog box.
    d. Enter a new name for the corresponding mapping.
    e. Click OK.
  6. Click Save.

Export Recipients to CSV

  1. Select Application > Notification > Recipients.
  2. The Recipients Editor tab displays.
  3. Select the Export tab.
  4. Expand Export to CSV and select the recipients to be exported and click to move the selection to the Selected for export text box.
    NOTE: Search recipients to narrow down the list by typing in the recipient name.
  5. Select a mapping configuration if present or create a mapping configuration. Refer to the Configuring Mappings for CSV Export section for more information.
  6. Click Export.
  7. The Save dialog box displays.
  8. Select the location where the CSV file must be saved.
  9. Enter a file name and click Save.
  1. The recipients are exported in CSV format.

Export Recipients to CSV using Command

  1. Select Applications > Notification > Recipients.
  2. The Recipients Editor tab displays.
  3. Select the Extended Operation tab.
  4. Click the Export CSV link to enable the export options.
  5. The Mapping, File Path field and Send button display.
  6. Enter the mapping to be used for the CSV export in the Mapping field.
  7. Select the File Path field, enter the local file path of the MNS server where the CSV file must be saved.
  8. Click Send.
  1. The recipients are exported in CSV format.

Export Recipients to CSV using Reaction

  1. Select Applications > Logics > Reactions.
  2. The Reaction Editor tab displays.
  3. Drag the recipients node from Applications > Notification > Recipients onto the Action expander of the Output expander.
  4. From the Property drop-down list, select Export to CSV [Commands.CSVExport.Request.Export].
  5. The ExportCSV[ExportCSV] command displays under Command column.
  6. Enter the mapping to be used for the CSV export in the Mapping field.
  7. In the ServerFilePath field, enter the local file path of the Notification server where the CSV file must be saved.
  8. Open the Triggers expander.
  9. From the drop-down list, select the appropriate logical operator (AND, OR).
  10. Open the Time and Organization Mode expander.
  11. Click Add to add a new time row and enter a new time or schedule that periodically triggers the export to CSV command.
  12. Click Save As and enter the name and description for the export to CSV reaction.
  1. The new export to CSV reaction displays in the System Browser.
