CS11 AlgoRex To FS20 Migration Limits and Compatibility

System Limits

The CS11 to FS20 Migration can support the replacement of fire objects in the following elements:

  • Graphics (including graphic commands, see however 3 – Complete the FS20 Configuration)
  • User-defined Views (other types of views are populated via import and not affected by the migration)
  • Scopes
  • Documents
  • Macros
  • Reactions (including triggers and outputs)
    NOTE: Macros and reactions cannot be migrated in case of different Object Models between source and destination panels. In such cases, it is not possible to safely match properties or values of source and target objects, and the log line indicates check properties and commands.
    A verification step is required in the migration procedure as described in 3 – Complete the FS20 Configuration).
  • Operating Procedures
  • Video Cameras
  • Related Items

The list of unsupported elements includes:

  • Scripts
  • Reports
  • Operator Tasks
  • Notifications
  • Trends
  • Schedules

Log Information

Automatically linked objects such as cameras, documents, and so on, do not display in the migration log because the links are dynamically resolved by Desigo CC and not directly migrated.

Compatibility Information

All supported CS11 and FS20/FS720 European versions are also supported by the migration procedure.

Distributed Systems

The migration procedure does not support multi-servers system (see Distributed Systems), and can only be applied on single local systems.
