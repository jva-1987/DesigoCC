SIPORT System Settings

On the SIPORT system, make sure to apply the configuration settings described in this section.

You can get technical information from SIPORT support (siveillance.support.industry@siemens.com), including the following set of preconfigured files:
- Door routines (K32R8000.txt file)
- Modbus mapping table (AttributesToModbus.txt file)

Firmware compatibility

The firmware version of the controllers must be V4.9-140822 or higher.

Configuration Workflow

  1. Configure reader and door parameters in the SIPORT hardware configuration client.
  2. Enable the Modbus interface. Proceed as follows:
    • In W3K32p.cfg, in the section [General], set the parameter ModbusIP=1
    • On all controllers, enable the Modbus/IP license
    • Apply a standard connection schema for all doors of your site. For example, Door 1: BI0100=door frame contact, BI0101 = door opener and so on. These addresses will then be used in the door routines.
  3. Configure and download door routines to collect the status of points and make all commands available. The attributes of the routines must be mapped to Modbus registers.
    NOTE: You can get an example of door routines from customer support (K32R8000.txt file). You may need to adapt it to the schema of your site.
    • In the SIPORT hardware configuration client, create a controller on address 8000.
    • Rename the K32R8000.txt file to K32R8000.386 and copy it to the Exos386D folder of the server.
    • Open the controller routine editor for the controller and check if the routines are available.
    • Once this template has been created, you can copy/paste the routines to all controllers with the default Import program.
    • Download the routines into the controller.
  4. Configure and download the Modbus mapping table.
    NOTE: You can get an example of Modbus mapping table from customer support (AttributesToModbus.txt file). For controllers and doors with access control, the list is limited to 000 => 089. For door supervision (customer specific solution), we use the addresses > 100. You may need to adapt the map to the configuration of your site.
    • Copy the content of AttributesToModbus.txt into the Modbus HMI tab in the SIPORT hardware configuration client.
    • Download the Modbus mapping table into the controller.
