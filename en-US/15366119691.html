Validation for Video

You can use validation to password-protect the video configuration against inadvertent or unauthorized modifications, and log any changes that are made to it. The standard validation mechanisms provided by Desigo CC are summarized in the table below. For more information see the section on validation. The specific validation mechanisms and behaviors that apply to video are described in the following sections.

Four Eyes Enabled

Camera and Monitor Controls

This refers to video-control actions on cameras and monitors done within the video application, such as selecting cameras, selecting preset positions, and opening or closing video streams.

You can set the validation profile separately on each camera, monitor, and other video objects. However, to get a consistent validation behavior when opening and closing a video stream, set the same validation profile on:

  • Cameras: opening a video stream is an action on the camera (drag-and-drop or selection).
  • Monitors: closing a video stream is an action on the monitor (clicking the X sign).
  • Camera groups and Camera sequences: these can be used to open video streams.

For these types of actions, there is a video-specific variant for the validation profile Enabled with Four Eyes disabled, highlighted in bold in the table below. It allows logging to happen without user input, by automatically inserting a fixed comment text instead. For all the other settings, the standard validation mechanisms apply.

Validation Profile

Effect for Video Control Commands

Four Eyes Disabled

Four Eyes Enabled

Monitored

No password requested, logged with fixed comment text, customizable in the Commanding Comment property.*
NOTE: This fixed comment behavior is not available on Flex client, where a user-entered comment is still required, as in the standard validation mechanism.

Supervisor name and password requested, logged with user-entered comment

Supervised

User password requested, logged with user-entered comment

User password +
Supervisor name and password requested, logged with user-entered comment

You can customize the fixed comment text as follows:

    • In System Browser, select Applications > Video.
    • In the Extended Operation tab, modify the text in the Commanding Comment property.

* This variant with fixed comment text applies only for video-specific actions. For other commands entered via standard Desigo CC UI functions, for example in the Contextual pane, the standard validation mechanism, with user-entered comment, continues to apply.

Video Step in Operating Procedures

For video control commands executed in operating procedures, the validation behavior is the same as that described in Camera and Monitor Controls, above. To support automated actions, such as starting a video recording, select a validation setting that does not require user input.

Configuration Changes Acquired from VMS

This refers to video configuration changes that are acquired from VMS, either automatically on Connect or through a Refresh command. For example, changing a camera from fixed to PTZ. For these modifications, the standard validation mechanisms are bypassed. The change is always logged without any user interaction, inserting a fixed comment text:

Validation Profile

Effect on Configuration Changes Acquired from VMS

Four Eyes Disabled

Four Eyes Enabled

Monitored

Irrespective of Validation/Four Eyes setting, no password is requested. Logged with fixed comment text (customizable in the Alignment Comment property).*

Enabled

Supervised

  • You can customize the fixed comment text as follows:
    • In System Browser, select Applications > Video.
    • In the Extended Operation tab, modify the text in the Alignment Comment property.

* This behavior with fixed comment text applies only configuration changes acquired from VMS. For other modifications entered via standard Desigo CC UI functions, for example in the Object Configurator tab, the standard validation mechanisms apply.
