Importing OPC Servers and OPC Items

Once you have scanned the OPC network for OPC servers you can import the selected OPC servers and their OPC items.

  1. System Manager is in Engineering mode.
  2. System Browser is in Management View.
  3. An OPC driver has been added, an OPC network has been added, and an OPC driver has been assigned to the OPC network.
  4. The computer for the OPC discovery has been set. See Configuring OPC DA Discovery Settings.
  5. OPC Servers and OPC Items have been discovered on the specified computer (see Discovering OPC Servers and OPC Items).
  6. To speed configuration, make sure the Desigo CC Transaction Mode is set to Simple or the Automatic Switch of Transaction Mode is set to True.
    The automatic switch setting is recommended. Without it, to re-enable the logs, you need to set the Transaction Mode to Logging at the end of the configuration procedure.
  1. In System Browser, select Project > Field Networks > [OPC network].
  2. In the Discovery tab, open the Device Discovery expander. For background information, see OPC DA Device Discovery.
  3. In the Network Scan section, verify the OPC servers selected for the import.
  4. (Optional) Select the row of a specific OPC server, click Objects and verify the relevant OPC items to import.
  5. Click Import.
  6. The Discovery status section displays the status of the operation. In particular, the Final Result column indicates the import result.
  7. When the import completes, the imported OPC servers and their OPC items are added to System Browser.
  8. (Optional) To view a summary of the import, click Import Log.
  1. The Import Log dialog box displays its outcome that you can also save in a .txt file.
