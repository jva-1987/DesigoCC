Importing Custom OPC Object Models into the OPC Library

When configuring the integration of a third-party OPC device for which the object models provided in the default OPC library are not sufficient, you can import custom object models from a configuration file. For background information see Object Model Import.

NOTE: You can skip this procedure if you plan to use default OPC object models.

  1. You are trained and authorized to work with system libraries. Also you are authorized to handle libraries for the specific customization level (L4-Project, L3-Country, or L2-Region).
  2. You prepared an OPC object model configuration file (CSV) which defines the custom object models that you require. See Create an OPC Object Model Configuration file (CSV).
  3. System Manager is in Engineering mode.
  4. System Browser is in Management View.
  1. Select Project > System Settings > Libraries > L2-Region or L3-Country or L4-Project > [Folder] > [OPC library] > [object model].
  2. In the Import tab, click Browse.
  3. In the Open dialog box, navigate to and select the CSV object model file that you want to use (for example, OPC Data Model Ver. 2.0).
  4. Click OK.
  5. The complete path of the selected file appears in the field. The Source Items preview displays all available items.
  6. Transfer the DPTs to import from the Source Items list to the Items to Import list .
  7. Click Import.
  8. A confirmation message displays.
  9. Click Yes.
  10. The Import tab displays the Import in progress page. When the import completes successfully, the Import dialog box displays its outcome.
  11. Click OK to close the dialog box.
  1. The DPTs contained in the CSV file that you selected as items to import are added to System Browser, under the object model folder.
