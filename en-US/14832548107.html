Starting or Stopping the OPC Driver

The OPC driver must be running (in Configuration Mode or Started) to connect to and import the configuration files. If Configuration Mode is used, when engineering is complete the driver must be Started to establish a real connection to the field. You may also need to stop and restart the driver for certain settings, such as changes to alarm tables, to take effect.

  1. System Manager is in Engineering mode.
  1. In System Browser select Management View.
  2. Depending on where the driver was created, select one of the following:
    - Project > Management System > Servers > Main Server > Drivers > [OPC driver]
    - Project > Management System > FEPs > [FEP station] > [drivers folder] > [OPC driver]
  3. In the Extended Operation tab, the Manager Status property indicates whether the driver is Stopped, Started, or in Configuration Mode.
  4. If the driver is Stopped, do one of the following:
    - Click Start Conf… to start the driver without establishing a real field connection.
    - Click Start to start the driver with a full connection to the field.
  5. If the driver is running in Configuration Mode, or if it is Started but you need to restart it:
    a. Click Stop.
    b. Click Start.
  1. The Manager Status changes to Started. The OPC driver is fully operational.
