OZW Troubleshooting

This section covers several areas of troubleshooting.

Troubleshooting SORIS

In SORIS, see SORIS Troubleshooting.

Handling OZW Error Conditions

Issue

Situation

Do the following...

In the Operation tab, the Online property of the OZW adapter under the Soris network indicates Disconnected.

The OZW adapter is not running or Desigo CC cannot reach the adapter.

  1. Check that the Windows service is running (Siemens OZW adapter).
  2. Check the adapter configuration: IP address and port number. In the Extended Operation tab, see the URL property of the OZW adapter. To modify the configuration, see Create the Adapter under the Network.
  3. From the Desigo CC station, check the IP connection to the adapter PC using Ping command.

In the Operation tab, the Connection Status property of the OZW device under the OZW adapter indicates Disconnected.

The adapter cannot reach the OZW device.

  • From the Adapter PC, check IP connection to the adapter PC using Ping command. From a web browser, try entering the OWZ IP address (as configured in the CSV file) in the address bar.

In graphics, the display of values is instable.

Standard cache setting does not fit the network connection

  • In the OZW Configuration CSV file, in the first line, modify the default cache value (30000) and try a higher number, for example 35000 or 40000.
