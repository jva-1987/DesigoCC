Configure Custom Import Rules

  1. Select Project > System Settings > Libraries > L4-Project > [Library Folder] > [Library Object] > [Import Rules].
  2. Open the Objects and Properties expander.
  3. Select an object model from the Objects section.
  4. (Optional) Configure the short name of existing objects.
  5. In the Address Profiles section, specify the address profile to be associated with the object model. By default, there will only be one profile however, you can create multiple profiles.
  6. Configure the following parameters for each listed property of the selected object model:
    - Direction
    - Function Code
    - Transformation Type
    - Offset
    - Absolute Offset
    - SubIndex
    - Poll Group Name
    - Visibility: Select the Visibility check box if you want the property of the selected object model to be displayed in the Operations and Extended Operations tab. By default, this check box is selected. If you do not want to display this property, you must deselect this box. create multiple profiles
    - Low Level Comparison: The Low Level Comparison check box is applicable only to those properties with the Direction field set to Input. By default, this check box is unchecked. If this box is selected, the driver sends the data only in case of changes.
  7. In the Message Conversion section specify the values for Min Raw, Max Raw, Min Eng, and Max Eng values. You can specify these values only to properties of type, PvssInt, PvssUint, PvssFloat, GmsInt, GmsUnit, and GmsReal.
  8. Click Save .
  9. Click OK.
  1. The import rules for the selected object model are configured.
