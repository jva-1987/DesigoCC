Setting Override Protection for Objects

Override protection prevents changes made manually to objects from being overwritten during a data import. For more information see Re-Importing Data and Override Protection. You can apply override protection to a single object or to a multi-selection of objects in System Browser.

Select Points to Be Protected

  1. In System Browser, select Logical View.
  2. From the drop-down list, select the display mode (Name, description).
  3. In the field, enter the name of the object for search, for example, setpoint.
  4. All matching objects are displayed in the System Browser.
  5. Select one or more objects. You can use CTRL or SHIFT to make multi-selections.
  6. The selected objects are highlighted in color and the number of selected objects is displayed in the Operation tab.

 

Set Override Protection for Objects

  1. In System Browser, select the object you want to protect. You can also select multiple objects as described above.
  2. Select the Object Configurator tab.
  3. In the Settings expander, select the Override protection check box.
  4. Click Save .
  1. The selected objects will be protected from being overwritten during a data import.

 

Clear Overrride Protection

  1. In System Browser, select the object you want to unprotect. You can also search for override-protected objects as described below.
  2. Select the Object Configurator tab.
  3. In the Settings expander, clear the Override protection check box.
  4. Click Save .
  1. The selected objects are no longer protected against being overwritten.

 

Search for Objects with Override Protection Set

  1. In System Browser, select Logical View.
  2. Click Filter search .
  3. The Filter search dialog box displays.
  4. Open the Other drop-down list.
  5. Select the Override Protection check box and then option Enabled.
  6. Click the gray background or entry fields to close the dialog box.
  7. Click Search.
  1. The found objects are displayed in the System Browser.
