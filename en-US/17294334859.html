Generate Report for Common Event Objects

Creating Own Report

The existing reports always acquire the entire building. Create your own reports for small partial plants that are tested.

Create a report

  1. The Common Event reports are imported.
  1. In System Browser, select Application View.
  2. Select Applications > Reports > State.
  3. The Default tab displays.
  4. Select one of the standard reports:
    • Common Event Objects in Fault
    • Common Event Objects in Alarm
    • Common Event Objects in Out Of Service (displays the state for Out of Service, Overridden, and Command)
  5. Click Save As .
  6. In the New Object dialog box, enter a name and description.
  7. Click OK.
  1. The new document displays in System Browser.

Defining Report Content

  1. The Common Event report displays.
  1. In System Browser, select Logical View.
  2. Select Logical > [network name] > [hierarchy level n].
  3. Drag the object to the report.
  4. Repeat Steps 2 and 3 on all objects covered by this report.

Defining Report Output

  1. The Common Event report displays.
  1. Select the Settings tab.
  2. For Report output click the symbol on the bottom right .
  3. The Report Output Definition opens.
  4. Select Report Format (for example, PDF) and define the destination type (for example, file).
  5. Click Configure Folders.
  6. The Report Output Folders Configuration dialog box opens.
  7. Enter the path and alias name of the target folder.
  8. Click Add.
  9. Click Close.
  10. In the Report Output Definition, click Add.
  11. The defined output is displayed in the List of Folders for Report Output.
  12. Click Close.
  13. Click OK.
  14. Click Save .
  1. The report is configured and is saved as a PDF file when running a reaction.
