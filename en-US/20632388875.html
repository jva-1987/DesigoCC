Configuring Event Rights

To allow visibility of events and commands in the management station and Flex Client, and the possibility to handle events with assisted treatment in the management station, users groups must be properly configured with appropriate scope, event rights, and application rights. For background information, see User Group Administration.

 

  1. System Manager is in Engineering mode.
  2. System Browser is in Management View.
  1. Open the Security application:
    a. Select Project > System Settings > Security.
    b. Select the Security tab.
  2. Select the user group:
    • To properly configure scope, event rights, and application rights, in the Groups list, select the appropriate user group. For example, OperatorsGroup.
  3. Assign scope:
    • To assign a scope for the selected user group, from System Browser, drag the appropriate scope (for example, OperatorsRoom) to the Scope field in the Scope Rights expander.
      NOTE: No other scope settings are required.
  4. Restrict event rights:
    • To disable event rights and the capability to issue commands, clear the Categories check box.
    • To restrict event visualization for a specific category, expand Categories, expand the desired category, and then clear the corresponding Show check box.
    • To restrict the capability to issue commands for a specific event category, expand Categories, expand the desired category, and then clear the check boxes for the commands to disable:
      - Acknowledge
      - Reset
      - Silence/Unsilence
      - Close
  5. Enable assisted treatment:
    • To enable assisted treatment for the selected management station user group, in the Application Rights expander, select Assisted Treatment.
  6. Click Save .

 

If Block Commands macros are used for handling fire events, then the corresponding commands must be enabled in the Event Rights expander for all event categories that a fire panel can handle.
If you want to hide an event category by disabling the corresponding event rights, you can disable the Show option only, and leave the required commands enabled: Acknowledge, Reset, and/or Silence/Unsilence, according to the Block Commands settings.
See Setting Up Block Command Macros.
