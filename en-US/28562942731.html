Multiple Desigo CC Connections

It is possible to connect up to thirteen (13) Desigo CC servers to the same SiPass server.

  • Each Desigo CC server must connect to SiPass through its own "SiPass RESTful Management Station API" service
  • One RESTful MS API is provided by default on the SiPass server. Additional ones may be installed--each on a different computer--as instructed in the SiPass Integrated RESTful MS API Installation Guide, provided in the SiPass integrated installation DVD.
  • The same SiPass user can be used for connecting from Desigo CC to SiPass through the different RESTful MS APIs.
  • A typical configuration is to install each additional RESTful MS API on the Desigo CC computer that will be using it to connect to SiPass.
    NOTE: The SiPass Operational Client and the .NET framework are prerequisites for installing the RESTful MS API. If installing the RESTful MS API on the same computer as Desigo CC, the SiPass client and API must be installed after installing Desigo CC

Multiple Desigo CC connections can be used to achieve the following solutions:

  • Separate Desigo CC servers operating independently (that is, not in distribution) can connect to the same SiPass server, for a redundant configuration:
  • In a distributed Desigo CC deployment, the interlinked Desigo CC servers can connect to the same SiPass server with distinct SiPass users, to partition the system based on the user rights. Each Desigo CC server will import only that portion of the SiPass subsystem for which its SiPass user has rights.

 

Operating System Requirements

Windows Server 2019 is supported only by SiPass V2.76 and higher.

Limitations in V2.70 only (Solved with V2.70 SP1 and higher)

  • Web page display in the access control step of operating procedures does not work with V2.70
  • Unsupported cardholder event transactions in V2.70:
    • Access denied - Programmable Authorization card
    • Access denied - Work group Capacity Full
    • Card holder did not enter
