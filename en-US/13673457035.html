Timeout Issues When S7 Points Are Exposed by the OPC Server

Message / Issue

Situation

Do the following...

Timeout issues in S7 devices whose data is exposed using the Desigo CC OPC DA server.

A project includes S7 devices whose points are exposed by the Desigo CC OPC DA server.

When a third-party OPC client connects to the Desigo CC OPC DA server, the values of S7 points are not correctly displayed. Also, the log files record timeout errors.

  • Navigate to [Installation Drive] > [Installation Folder] > GMSMainProject > bin and openthe file Siemens.Gms.OPCServer.Settings.txt.
  • In the [DeviceToSupport] tag, enter S7 for DeviceType and save the changes.
  • If the OPC server is running, stop and restart it. See Stopping the OPC DA Server and Start the OPC DA Server.

NOTE: The default value of DeviceType is PX. If DeviceType is empty, invalid, or the [DeviceToSupport] tag is not present, the behavior will always be the one expected for PX.
