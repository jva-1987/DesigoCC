Intrepid RPM Troubleshooting

This section covers several areas of troubleshooting.

Troubleshooting SORIS

In SORIS, see SORIS Troubleshooting.

Handling Connection Error Conditions for Intrepid RPM

Issue

Situation

Do the following...

The Intrepid RPM adapter cannot reach the Intrepid RPM website.

The connection is lost because your internet connection or the Intrepid RPM website is down.

  1. Check your internet connection.
  2. Check the Intrepid RPM website connection:
    a. In System Browser, select Project > Field Networks > [SORIS network] > [Intrepid RPM adapter] > Configuration > Adapter Settings.
    b. In the Operation tab, check the Service Status property. If Disconnected, wait until the connection is re-established and the Service Status property becomes again Connected.

In the Operation tab, the Service Status property for the adapter indicates Disconnected.

The adapter is not running or the connection between Desigo CC and the configured URL is not working.

  1. Check if the adapter is running.
  2. Check the IP address and the port number.

After the adapter configuration and discovery are carried out, the objects are not created in System Browser.

The configuration file is invalid.

Check the configuration file:

  • Check that the Intrepid RPM adapter Config.json file is located in the same directory as the Intrepid RPM adapter executable file.
