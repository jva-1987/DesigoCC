Advanced CAD Importer Troubleshooting

Message / Problem

Situation

Do the following...

The Import Summary dialog box indicates import warning or failures:

Graphics imported with warnings: …

Graphics failed to import: …

Errors occurred in importing some maps.
Warnings indicate that errors could be recovered. Instead, import failed messages mean that one or more maps could not be imported at all.

See the detailed log files:

[drive]:\Users\[username]\AppData\Local\Temp\AdvancedCadImport_mm-dd-yyyy_nn.txt
and

[drive]:\Users\[username]\AppData\Local\Temp\AdvancedCadImport_Results yyyymmdd hhmmss.txt

The detailed log indicates:

Unable to map symbol

The object specified in a CAD block attribute could not be found in Desigo CC.

Check and correct the block/attribute settings in Select the Object Blocks and Identification Attributes.

The detailed log indicates:

Error retrieving symbol

No symbol could be found in the selected style for the imported object.

Check and correct the symbol selection in Select the Symbol Style, Object-Match Criteria, and other options. Alternatively, you may need to check the library configuration for the graphic symbols and their association to functions and object model.

The detailed log indicates:

Duplicated symbol found

The same object was found in multiple CAD blocks. Only the first object found can be imported.

Correct the CAD drawings and import again.
The tool requires no object duplication in the CAD drawings. This is necessary to identify objects unambiguously in subsequent imports.

The detailed log indicates:

Invalid X or Y position

The symbol was placed in position “0,0” (top left) because of invalid coordinates.

From graphic editor, check and correct the symbol in the imported graphic.

The detailed log indicates:

Invalid height or width

The symbol dimensions were not correct, for example one of them is =0.

From graphic editor, check and correct the symbol in the imported graphic.

Symbols do not change in the graphics after re-importing with a new style setting.

You want to change symbol style, but no symbol reference is modified by re-imports.

Change of style is not allowed. You can delete the imported graphics and start a new import with the correct symbol style. Alternatively, from graphic editor, you need to replace each symbol instance in the graphics.

In virtual machine environments, an error occurs during complex map creation, especially with Variable Resolution or Very High Quality settings:

Error creating the AutoCAD Object for the graphic …

The graphic card settings in the virtual machine environment may not be properly set to handle complex AutoCAD images. Specifically, the 3D support is not required and can be disabled to recover the necessary resources.

In virtual machine environments, for example VMware vSphere ESXi, in the Graphic Card settings, deselect the Enable 3D Support option.

The imported CAD drawing is not properly imported, and the resulting image is somehow cut.

If a virtual model space viewport is used, the imported may fail to fully import the image. See also Advanced CAD Importer Compatibility and Limits.

Before importing, replace virtual model space viewports by regular paper space viewports.
