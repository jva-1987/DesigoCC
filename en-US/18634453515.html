Setting CS6 Guarto Configuration From Scratch

If you have the default configuration (CSV) file that does not contain configuration data, you must prepare the configuration from scratch. For background information, see CS6 Guarto Integration.
NOTE: To modify the object function mapping, see Adjusting the Object Function Mapping.

Do the following procedure only if you do not have a (CSV) file with CS6 configuration data to load. Loading the CS6 configuration is highly recommended instead. For instructions, see step Loading CS6 Configuration.

 

  1. After installation, the application Siemens.GMS.CS6.Configurator is available in AdditionalSW\CEI CMSDL CS6 Configurator locally or over the network.
  2. The CS6 adapter is running and connected to the CS6 intrusion panels. For instructions, see Installing and Starting the CS6 Guarto Adapter.
  1. Set the protocol type:
    a. In the file system, locate AdditionalSW\AdditionalSW\CEI CMSDL CS6 Adapter\Files.
    b. Open the ConfigurationSettings.xml file with a text editor (for example, Notepad), and specify <ProtocolType> in one of the following ways:
    - 1 (CMSDL)
    - 0 (CEI-ABI)
    c. Save the changes.
  2. Start the configurator tool.
    • Double-click the application Siemens.GMS.CS6.Configurator.
  3. The Siemens.GMS.CS6.Configurator tool page is displayed.
  4. Load the connection parameters from the configuration (XML) file:
    a. On the Siemens.GMS.CS6.Configurator tool page, select System on the right.
    b. In the Branch XML file connection parameters section, click .
    c. Browse the file system and reach CEI CMSDL CS6 Adapter\Files.
    d. Select the CEIProtocolConfiguration.xml file.
    e. In the From file section, click Load.
  6. Manually set a new branch.
    • Click Add new branch . Then in the Load new branch dialog box, enter Description, ID, IP, and Port.
  7. The newly created branch is displayed under System. Its status is (Offline).
    NOTE: You can later modify this configuration. For instructions, see Adjusting CS6 Configuration.
  8. Manually add as many subsystem data structures as required. Do the following:
    a. Select the branch, and click Add subsystem .
    b. In the Load new subsystem dialog box, enter Description and ID, and click Load.
    NOTE: Enter the exact ID used by the CS6 panel.
  9. The subsystem data structure is created under the selected branch. For instructions on how to set CS6 configuration data, see Manually Setting Subsystem Items.
  11. Save the configuration. Select System on the right, and in the Configuration area, do one of the following:
    • Export the configuration to a CSV file:
      a. In the To File section, click Export.
      b. Click Yes.
      c. In the Browse for Folder dialog box, select AdditionalSW\CEI CMSDL CS6 Adapter\Files as location where you want to save the files CEIProtocolConfiguration.xml, CMSDLProtocolConfiguration.xml, and FieldConfiguration.csv, and click OK.
      NOTE: If you already saved the CSV configuration file at this location, the existing CSV configuration file is overwritten with the new configuration.
    • Override online data:
      a. Connect to the adapter. For instructions, see Connecting the Configurator Tool to the Adapter.
      b. In the To Adapter section, click Download.
