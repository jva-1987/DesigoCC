Enable Incident Triggering

Notification can send notifications for events, for example, a fire alarm in Building A, depending on the time, date, type of day (working day or weekend) and the organization mode. This is possible using the incident triggering command functionality.

This section describes procedures to enable incident triggering.

Enable Incident Triggering using Reaction

  1. Select Applications > Notification > Incident Templates > [incident template].
  2. In the Operation tab, set the Incident Triggering command to True.
  3. Select Applications > Logics > Reactions.
  4. The Reaction Editor tab displays.
  5. Open the Output expander and thereafter, open the Action expander.
  6. From System Browser, drag the incident template, for example, fire trigger, into the Action expander.
  8. From the Property drop-down list, select Incident Triggering [IsTriggeringEnabled] to configure the trigger for the selected incident template.
  9. From the Command drop-down list, select Set [EnableIncidentTriggering].
  10. Set the value of the IsTriggeringEnabled command to True in the check box that displays below the Command drop-down list. To disable the incident triggering, set the value of this command to False.
  11. In the Triggers expander, specify if the conditions specified in both, Events as well as Values and States, expanders must be true or if the conditions specified in either of the expanders must be true by selecting the appropriate operator (AND, OR) in the drop-down list.
  12. Open the Time and Organization Mode expander.
  13. Click New to add a new row and specify a new time or schedule, when you want to trigger the Initiate Incident command.
  15. Click Save As and enter a name and description for the incident triggering reaction.
  1. The new incident triggering reaction is added to System Browser.

Enable Incident Triggering for a Specific Period using Reactions

You want to enable the ‘University Update' incident to send out notifications from the university. It is expected that the Notification system triggers University Update incidents only during the 1st shift from 6.00 A.M to 6.00 P.M.

  1. The incident template University Update is created and configured.
  1. Select Applications > Notification > Incident Templates > [incident template].
  2. In the Operation tab, set the Incident Triggering command to True.
  3. Select Applications > Logics > Reactions.
  4. The Reaction Editor tab displays.
  5. In the Time and Organization Mode expander, set the value of Set Start Time and Set End Time to 6.00 A.M. and 6.00 P.M. respectively.
  6. Open the Output expander and thereafter, open the Action expander.
  7. From System Browser, drag the University Update incident template, into the Action expander.
  8. From the Property drop-down list, select Incident Triggering [IsTriggeringEnabled] to configure the trigger for the University Update incident template.
  9. From Command drop-down list, select Set [EnableIncidentTriggering].
  10. Set the value of the IsTriggeringEnabled command to True in the check box that displays below the Command drop-down list. To disable the incident triggering, set the value of this command to False.
  11. From System Browser, drag the University Update incident into the Else Action tab.
  12. Set the IncidentTriggering command to False.
  1. The incident University Update is available for triggering only in the specified time period.

Enable Incident Triggering using Command

  1. Select Applications > Notification > Incident Templates > [incident template].
  2. In the Operation tab, set the Incident Triggering command to True if you want to enable the incident trigger for the selected Incident. Set the value to False to disable the trigger. If set to False, for an incident, then the incident is not initiated even when an event matching its trigger rule occurs in the system.
  3. Click Set.
  1. The selected incident template is initiated when any event matching its trigger rule occurs in the system.
    NOTE: In order to view the command results after command execution, you need to refresh the user interface by re-selecting the incident template from System Browser.

 

Enable or Disable Triggers for Notification

You can enable or disable triggers at project level.

  1. System Manager is in Engineering Mode.
  2. The Notification project level node is selected.
  1. In the System Browser, select Application View.
  2. Select Application > Notification.
  3. Select the Extended Operation tab.
  5. In Incidents Triggering, select the Enable or Disable buttons to enable or disable the trigger respectively.
  1. The trigger status displays in the Incident Triggering section.
