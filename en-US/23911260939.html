Management Station (MS) Alarms

Management station alarms are instead defined directly in the Desigo CC management platform. This type of alarming requires an engineered project and should only be used as an exception. Wherever possible, define alarming on the automation station.

Management station alarms cannot be forwarded if Desigo CC is not online.

The following alarm priorities are available:

  • Life Safety
  • Alarm
  • Supervision
  • General
  • Maintenance
  • Trouble
  • Fault
  • Status

Set up a Discrete Digital Alarm

  1. You want to configure a discrete alarm on a Boolean property that has the Boolean text group assigned.
  1. In System Browser, select Project > System Settings > Libraries > [... ] > [object model].
  2. Select the Models & Functions tab.
  3. In the Properties expander, select the Boolean property on which to configure the alarm.
  4. Open the Alarm Configuration expander.
  5. Select the Valid check box to enable the Alarm Configuration fields.
  6. Select the Management Station option.
  7. The alarm table displays underneath.
  8. From the Alarm kind drop-down list, select Discrete.
  9. In the Normal row, enter texts for Event Text and Normal Text.
  10. In the second row:
    a. Click the Alarm Class field and, from the drop-down list, select the status, for example Alarm.
    b. In the Value Range column, select the operand and the related state.
    c. Enter the texts for Event Text and Normal Text.
  11. Select the Alarm config. activated check box.
  12. Click Save .
  13. The Boolean alarm is created on the management platform. The corresponding MS check box is selected in the Properties overview.

 

Set Up a Continuous Digital Alarm

  1. You want to configure a continuous alarm on a Boolean property that has the Boolean text group assigned.
  1. In System Browser, select Project > System Settings > Libraries > [... ] > [object model].
  2. Select the Models & Functions tab.
  3. In the Properties expander, select the Boolean property on which to configure the alarm.
  4. Open the Alarm Configuration expander.
  5. Select the Valid check box to enable the Alarm Configuration fields.
  6. Select the Management Station option.
  7. The alarm table displays underneath.
  8. From the Alarm kind drop-down list, select Continuous.
  9. In the Normal row, enter texts for Event Text and Normal Text.
  10. In the second row:
    a. Click the Alarm Class field and, from the drop-down list, select the status, for example Alarm.
    b. In the Limiting Value column, select the operand and the related state.
    c. Enter the texts for Event Text and Normal Text.
    d. (Optional) Enter an Upper and Lower Hysteresis.
  11. Select the Alarm config. activated check box.
  12. Click Save .
  13. The Boolean alarm is created on the management platform. The corresponding MS check box is selected in the Properties overview.

 

Set up an Analog Management Station Alarm

  1. You want to configure a continuous alarm on a property.
  1. In System Browser, select Project > System Settings > Libraries > [... ] > [object model].
  2. Select the Models & Functions tab.
  3. In the Properties expander, select the property on which to configure the alarm.
  4. Open the Alarm Configuration expander.
  5. In the Alarm Configuration expander, select the Valid check box.
  6. Select the Management Station option.
  7. Click New.
  8. A new line is added to the table.
  9. From the Alarm Class drop-down list, select the status.
  10. In the Normal row, enter texts for Event Text and Normal Text.
  11. In the second row, from the drop-down list, select the status, for example Alarm.
    a. In the Value Range column, select the operand and the related values.
    b. Enter the texts for Event Text and Normal Text.
  12. Repeat the appropriate entries for each state.
  13. Select the Alarm config. activated check box.
  14. The analog alarm is created on the management platform. The corresponding MS check box is selected in the Properties overview.
  15. Click Save .

 

For Uint64 data types, if you configure a discrete alert with a range with maximum higher upper limit (that is, specifying the Max Uint 64: 18446744073709551616), you will only see an event if the property value is 2^63 - 1 (9223372036854775807) or lower. Any higher value of the property will not trigger the alert.
