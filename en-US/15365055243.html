Installing the VMS Server

The Milestone / Siveillance Video Management System (VMS) server can be installed on the same computer as the Desigo CC server, or on a separate computer. For details, see Video Architectures and Limits. Complete one of the below procedures, depending on where the VMS server will be installed.

 

Prerequisites

  • You have the VMS server installer file.

 

Option 1: Install VMS Server on a Separate Computer from the Desigo CC Server

Do these steps on the separate computer that will host the VMS server.

  1. Install the VMS server following its separate installation instructions. In the setup wizard, you can select the Single Computer (recommended) or the Custom installation variant.
  2. On the Select Encryption page, if you turn on encryption of Streaming media, the CA certificate that signed the selected VMS SSL Certificate must be installed on all Desigo CC client stations that display video streams.
    You can also enable encryption after installation from the Server Configurator. For more information, see Encrypting Video Communication Flows.

NOTE: A configuration where Desigo CC and VMS are installed on different machines belonging to different domains is not supported.

 

 

Option 2: Install VMS Server on the Same Computer as the Desigo CC Server

Do these steps on the Desigo CC server computer.

NOTICE

Starting from Milestone/Siveillance v13.1a 2019 R1, installation of the VMS will fail on a computer where the Desigo CC server is already installed. Accordingly, the VMS software must be installed before you install Desigo CC with the Video extension.

  1. Do one of the following:
    • For Milestone/Siveillance v13.1a 2019 R1 or later, check that Desigo CC is not yet installed on the computer. If it is, you must uninstall Desigo CC and its prerequisites, and turn off the WebDAV Publishing Windows feature.
    • For earlier versions of Milestone/Siveillance, if Desigo CC is already installed on the computer, check the default ports and website.
  2. Install the VMS server following its separate installation instructions. In the setup wizard you can select:
    • The Single Computer installation variant, which is largely predefined and very easy (you can only select the file location and the product language).
    • The Custom installation variant (recommended to optimize the SQL database functions).
  3. If you chose the Custom option, in the SQL section, do one of the following:
    • If Desigo CC is already installed, select the GMS_HDB_Express SQL server instance.
    • If the VMS software is being installed before Desigo CC, you can also share the SQL Server installed by the VMS with GMS.
      NOTE: Use the default settings for the remaining custom installation options.
  4. On the Select Encryption page, if you turn on encryption of Streaming media, the CA certificate that signed the selected VMS SSL Certificate must be installed on all Desigo CC client stations that display video streams.
    You can also enable encryption after installation from the Server Configurator. For more information, see Encrypting Video Communication Flows

 

 

Check the Default Ports and Website

If you are installing the VMS server on a computer where the Desigo CC server is already installed (this is possible only for Milestone/Siveillance versions prior to v13.1a 2019 R1), you must perform these steps to resolve possible conflicts.

Default ports and website
The VMS uses default website ports 80 (HTTP) and 443 (HTTPS) for its client stations. This setting cannot be changed. When installing VMS on the same computer as the Desigo CC server, you must check in SMC that these ports are not already used by a Desigo CC website (for example, to support Web Services), and if necessary switch the Desigo CC website to use different ports (for instance, 8080 and 8443, respectively). In addition, if the default IIS website was removed during the Desigo CC installation, you must re-create it before installing VMS.

  1. In the SMC Websites, do not use the ports 80 and 443, which will be used by VMS.
  2. If the default IIS default web site was removed (for example to replace it with a Desigo CC website), you must create it again before installing the VMS server.
    a. Run the Internet Information Services (IIS) Manager (refer to Microsoft documentation)
    b. In the Sites node, re-create the default web site.
    c. In the Add Web Site dialog box, enter the following settings:
    • Site name: enter Default Web Site
    • Application Pool: click Select... and choose DefaultAppPool
    • Physical path: enter %SystemDrive%\inetpub\wwwroot
    • Binding options: use the default settings
