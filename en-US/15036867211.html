Fire Brigade Maps Generator Troubleshooting

Message

Situation

Do the following...

The Import Summary indicates warning or failures:

Maps processed with warnings: …

Maps processed with errors: …

Problems occurred in creating some maps.
Warnings indicate that problems could be recovered. Instead, errors indicate that one or more maps could not be created at all.

See the detailed log files:

<drive>:\Users\<username>\AppData\Local\Temp\FireBrigadeMaps_mm-dd-yyyy_nn.txt
and

<drive>:Users\<username>\AppData\Local\Temp\FireBrigadeMaps Results yyyymmdd hhmmss.txt

Fix the possible errors in the template or in the graphics using the management platform tools. Refer to Fire Brigade Maps Template and Preparing Graphics for Fire Brigade Maps.

One or more of the PDFs generated from the Fire Brigade Map have the map barely visible: the map lines are too thin and blend into the background.

In the original graphic, plan lines are too thin and the generated PDF accentuates this aspect.

In Graphics Editor Element Tree, select Layer > XPS. In the XPS properties, increase the value of the Stroke thickness factor until the result is acceptable.
Once the PDF generation is complete, if you want to return to an optimal map display in Desigo CC, remember to reset the Stroke thickness factor to its original value. Refer to Graphics Editor Workspace.
