TECLA Script

Scenario

You want to migrate TECLA scripts.

The process of migrating the TECLA Scripts to JavaScript™ broadly involves three activities:

  1. Exporting the TECLA Script from SICLIMAT X OS storage
  2. Importing the TECLA Script in Desigo CC
  3. (Optional) Modifying and extending the migrated scripts

For general information on TECLA scripts, see SICLIMAT X TECLA Scripts.

Export Repository from SICLIMAT X OS

The process of exporting engineering data from SICLIMAT X OS is a common step during the migration of SICLIMAT X OS to Desigo CC. A routine called the Repository Extraction Tool in SICLIMAT X OS extracts all required information to a structure of folder and files. You can find more information about this tool under SICLIMAT X Export. For the migration of TECLA scripts, there are two essential parts – the TECLA script documents and the data point information in an XML file.

 

Import Scripts into the System

  1. The relevant data points of Siclimat are imported. For more information, see Importing the SICLIMAT X Project Data Files).
  2. System Manager is in Engineering mode.
  3. An S7 network exists.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Conversion Tools > Siclimat Script Migration.
  3. Select the Script Migration tab.
  4. From the Select network drop-down list, select S7Network.
  5. Click Browse .
  6. In the Repository Folder Selector dialog box, select the folder.
  7. The scripts are listed.
  8. Select the check boxes of the scripts to migrate.
  9. Click Run .
  10. The execution status is updated.
  11. The Import summary dialog displays.
  12. Click OK.
  1. The data point objects representing the scripts and their content are available in System Browser.
