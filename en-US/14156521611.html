Configuring Import Rules for a Manually Created OPC Library

  1. You are configuring a manually-created OPC library for integrating a third-party OPC device into Desigo CC.
  2. You are trained and authorized to work with system libraries. Also you are authorized to handle libraries for the specific customization level (L4-Project, L3-Country, or L2-Region).
  3. You manually created an OPC library for a device (see Manually Creating an OPC Library).
  4. If required, you imported custom OPC object models into that library (see Importing Custom OPC Object Models into the OPC Library).
  5. System Manager is in Engineering mode.
  6. System Browser is in Management View.
  1. Select Project > System Settings > Libraries > L2-Region or L3-Country or L4-Project > [Global] > [OPC library] > [import rules].
  2. Select the Import Rules tab.
  3. In the Object and Properties expander, specify the import rules that map the OPC types with OPC object models:
    a. Click Add Object, to add a new row.
    b. Edit an OPC Type and select the corresponding Object Model.
    c. Select a value for the following fields: Direction, Transformation Type.
    d. Repeat the previous steps for any required objects.

 

The import rules defined in the standard OPC library are optimal for OPC integration. When configuring new OPC import rules, we suggest copying the settings from the OPC import rules provided for the L1-Headquarter > Global > OPC library (make a screenshot and copy from the image). Even though you can freely define the mapping, the configuration of L1-Headquarter can be considered consistent and working with the majority of the OPC object models.
