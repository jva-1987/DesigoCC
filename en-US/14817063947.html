Meteoblue Weather Troubleshooting

This section covers several areas of troubleshooting.

Troubleshooting SORIS

In SORIS, see SORIS Troubleshooting.

Handling Connection Error Conditions for Meteoblues Adapter

Issue

Situation

Do the following...

The Meteoblue adapter cannot reach the Meteoblue website.

The connection to the Meteoblue weather forecast service is lost because your internet connection or the Meteoblue website is down.

  1. Check your internet connection.
  2. Check the Meteoblue weather forecast connection:
    a. In System Browser, select Project > Field Networks > [SORIS network] > [Meteoblue adapter] > Configuration > Adapter Settings.
    b. In the Operation tab, check the Service Status property. If Disconnected wait until the connection is re-established and the Service Status property becomes again Connected.

In the Operation tab, the Service Status property for the adapter indicates Disconnected.

The adapter is not running or the connection between Desigo CC and the configured URL is not working.

  1. Check if the adapter is running.
  2. Check the IP address and the port number.

After the adapter configuration and discovery are carried out, the objects are not created in System Browser.

The configuration file is invalid.

Check the configuration file:

  1. Check that when you select Adapter Settings in System Browser, the configuration file full path is indicated next to the Configuration File property in the Operation tab.
  2. Check that the meteoblueadapter.conf file is located in the same directory as the Meteoblue adapter executable file.

How to Reset the Secure Key

Scenario: For security reasons, you have to reset the secure key code.

  1. In System Browser, select Project > Field Networks > [SORIS network] > [Meteoblue adapter] > Adapter Configuration.
  2. Next to the Secure Key property, enter -9999 in the field and click Change.
  1. The Secure Status property becomes Disabled.
