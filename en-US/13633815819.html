Manually Modifying the Logical or Physical View

You can modify or reorganize the structure of the logical view or the physical view.

Prerequisites

  • System Manager is in Engineering mode.
  • System Browser is in Management View.

Activate the State Propagation from the Child Nodes to the Parent Nodes in a View

  1. Select Project > System Settings > Views > [logical view] or [physical view].
  2. The views workspace displays in the Views tab.
  3. Select the Enable propagation check box.
  4. Click Save .

 

Modify the Structure of Levels (Aggregators)

To avoid mismatch between Desigo CC views and the content of subsystem configuration files, it is not recommended to manually modify the structures of aggregators in Desigo CC views that are automatically created by importing those configuration files.
In case a Desigo CC view no longer reflects the subsystem hierarchy provided in the configuration file, you must manually remove any of the nodes manually added to this view and then proceed to re-importing the configuration file. Note that deleting entire subtrees in a view might result in links to maps, reactions, and so on being lost.

 

  1. Select [logical view] or [physical view] from the System Browser drop-down list.
  2. In the tree structure, select the root node of the view.
  3. The view builder workspace displays in the Views tab.
  4. Click to expand the view structure.
  5. In the Views tab, do one or more of the following:
    • Add new aggregator:
      a. Select the node where you want to add a new aggregator. You can only add aggregators to the view root, or to other aggregators.
      b. Click Add Aggregator.
      c. In the New Object dialog box, enter a name and description and click OK.
      d. Repeat the preceding steps to add more levels. You can add nested levels to create a hierarchical structure.
    • Use Rename Node to change the description of any link or aggregator.
    • Use Cut/Copy/Paste Aggregator to rearrange the levels.
    • Use Remove Node to delete unwanted levels.
  6. Click Save .

 

Populate the Levels with Links to System Objects

  1. Select the Manual navigation check box.
  2. Select [logical view] or [physical view] from the System Browser drop-down list.
  3. In the tree structure, select the root node of the view.
  4. The view builder workspace displays in the Views tab.
  5. If you want to link a subtree (object plus all its children), in the Views tab, select the Link children check box. If you want to link only an individual object, in the Views tab, clear the Link children check box.
  6. In System Browser, select the object that you want to link and drag it to the desired level in the Views tab.
  7. Repeat the preceding steps to link more objects or subtrees.
  8. (Optional) Use Remove Node to remove unwanted links.
  9. Click Save .

 

Manually Change the Sort Order of Sibling Nodes

  1. Select [logical view] or [physical view] from the System Browser drop-down list.
  2. In the tree structure, select the root node of the view.
  3. The view builder workspace displays in the Views tab.
  4. To pin a child node in a subtree, in the Views tab select the desired child node, click Pinned Node, in the New position field, set a value corresponding to the new position for the child node, and click OK. To view the values of the resulting order, on the top left, select the Show nodes order check box.
  5. To change the nodes position in a subtree, in the Views tab, select one or more nodes that you want to move, and use Move top, Move up, Move down, or Move bottom to change their position.
  6. Repeat the preceding steps to change the sort order of more nodes.
  7. Click Save .

 

Restore the Default Sort Order of the Sibling Child Nodes in a Subtree

  1. Select [logical view] or [physical view] from the System Browser drop-down list.
  2. In the tree structure, select the root node of the view.
  3. The view builder workspace displays in the Views tab.
  4. Select the parent node of the subtree whose sort order you want to reset.
  5. Click Reset children order.
  6. Click Save .

 

Remove Any Empty Aggregators

  1. Select [logical view] or [physical view] from the System Browser drop-down list.
  2. In the tree structure, select the root node of the view.
  3. The view builder workspace displays in the Views tab.
  4. Select the root node of the view.
  5. Click Purge Aggregators .
  6. Click Save .

 

Update the View to Reflect the Changes in Linked Objects

  1. Select [logical view] or [physical view] from the System Browser drop-down list.
  2. In the tree structure, select the root node of the view.
  3. The view builder workspace displays in the Views tab.
  4. Click Update View .
  5. Click Yes.
  6. When the update is complete, the tree structure is collapsed in the preview area.
  7. Expand the tree structure in the preview area.
  8. Any soft links relevant to new elements linked to the tree structure display in green, while broken links display in red.
  9. To remove broken links, select the soft links in red and click Remove.
