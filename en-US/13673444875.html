PX Quality Flag Issue When PX Points are Exposed by the OPC DA Server

Message / Issue

Situation

Do the following...

PX bad quality data.

A project includes a PX device whose points are exposed by the Desigo CC OPC DA server.

The first time the project is started and the first third-party OPC client connects to the Desigo CC OPC DA server, the data quality flag of all the PX points is set to bad. No error message displays. No errors are recorded in log files.

When the next third-party OPC client connects, the quality flags display correctly.

  • Navigate to [Installation Drive] > [Installation Folder] > GMSMainProject > bin and openthe file Siemens.Gms.OPCServer.Settings.txt.
  • Check that in the [DeviceToSupport] tag PX is specified for DeviceType. If not, enter this text and save the changes.
  • If the OPC DA server is running, stop and restart it. See Stopping the OPC DA Server and Start the OPC DA Server.

NOTE: The default value of DeviceType is PX. If DeviceType is empty, or the [DeviceToSupport] tag is not present, the behavior will always be the one expected for PX.
