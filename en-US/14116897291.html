Copying a Graphics Page

The project must be considered as a whole to efficiently engineer the graphics. You may, for example, decide to start with the largest plant, rather than the first ventilator in the building. This can then be copied and modified as a template for all other ventilation plants. The graphic references for a copied graphics page can then be referenced using:

  • Find and Replace
  • Reassign individual references
  1. Open the graphics page to be copied.
  2. Click Save As .
  3. Select the appropriate graphic folder and in the Name field, enter a new file name.
  4. Click OK.
  1. The graphics page is copied and saved under a new name.

 

Modifying Object References using Find and Replace

  1. You want to delete unneeded graphic objects for the new ventilation plant.
  1. In the View tab select Find and Replace.
  2. The Find and Replace dialog box opens.
  3. In System Browser, select the corresponding hierarchy in the object tree, for example, Logical > [Network name] \...\ [Plant].
  4. Drag the object from the existing plant to the Find what text field.
    For example, System1.LogicalView:Logical.Desigo.B01.APlt.Ahu01.
  5. Delete the ; character at the end of the object reference.
  6. The found object references are displayed in the Find Result dialog box.
  7. Drag the object from the new plant to the Replace with field.
    For example, System1.LogicalView:Logical.Desigo.B01.APlt.Ahu02.
  8. Apply the default settings for the Find what and Replace with fields.
  9. Click Replace All.
  10. The object references are replaced. No further object references are displayed in the Find Results dialog box.
  11. Click Save .
  12. Continue graphic engineering for this graphics page by:
    • Changing the plant name (this is normally the title of the text in the center of the page)
    • Modifying existing graphic links to other graphics pages
    • Adding missing objects
    • Modifying object reference for the outdoor sensor. Reason: The outdoor sensor is generally not in the same structure as the plant.

 

Individually Modifying Object References

This procedure is particularly helpful if an existing graphics page requires only a few data points from another object hierarchy. The outdoor sensor typically exists only once in a project, but must be displayed on various graphics pages.

  1. The outdoor sensor exists on the graphics page with an incorrect object reference.
  1. In System Browser, select Logical view.
  2. Select Logical > [Network name] \...\ [Plant] > [TOa (outdoor sensor)].
  3. Drag the object reference to the symbol or in the Symbol field Properties > Symbol Instance > Object Reference.

 

Copying Across a Project

Existing graphics for a project are copied to a new one and modified.

  1. The Graphics Editor is opened.
  2. No graphics page is open.
  1. Open Windows Explorer and select Source project > Graphics.
  2. Select the graphics page to be copied (.ccg).
  3. Drag the page to Start > Edit.
  4. The page is copied and displayed.
  5. Click Save As .
  6. Select the appropriate graphic folder and in the Name field, enter a new file name.
  7. Click OK.
  8. The graphics page is copied and saved under a new name.
  9. Edit the copied graphic and adapt the object references using Search and replace or Individually.

NOTE:
Incorrect object references are displayed in Test- or Online mode using #ENG.
