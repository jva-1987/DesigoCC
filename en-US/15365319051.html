Disabling Video Tag Commands

Scenario: On some VMS systems, the tag commands do not automatically start the video recording. To prevent users from inadvertently creating tags with no associated video recording, you may want to make the tag commands unavailable when video recording is not active. For background information see Recording Video, and Milestone / Siveillance VMS Reference.

This modification involves the system video library and will apply to all cameras. When video recording is not active on a camera, the corresponding tagging buttons will display as dimmed.

NOTICE

System malfunction

Incorrect modifications to system libraries will cause the product to malfunction.
Do not modify system libraries unless you have enough knowledge and experience to change them without destroying or corrupting crucial data.

  1. You are authorized and trained to customize the system libraries.
  2. System Browser is in Engineering mode.
  1. Access the Video library and customize the Camera object model.
  2. In the customized object model, select the Status.Recording properties.
  3. In the Command Configuration expander, in the first available Dependency field, select the Status.Recording property.
  4. Click Save .
  5. A new column displays in the command list to set a dependency from the recording status.
  6. In the Creating Manual Video Tag line, set the Status.Recording dependency equal to True.
  7. Click Save .
