Handling Import Error Conditions

Message

Situation

Do the following…

Invalid file format

The selected file is not a valid database file from the tool.

Create a new export file and try importing it.

Import failed – Unable to create a new data point.
Abort, Continue, Continue All

The import did not recognize one of the objects for import.

If you do not know the exact reason for the missing object type, click Abort to stop the import procedure.

Alternatively, you can choose to Continue and skip the missing object or click Continue All to ignore any further recoverable errors.

In any case, try creating a new export file and importing it. If the problem persists, contact Technical Support.

Multiple networks are present.

The import zip file contains devices from more than one network.

Select the required network to import the data points.

Import not possible. Import data would exceed system limits.

The import function recognized more points than defined on the project. After clicking OK, the import is canceled.

The project is set to Small by default and supports 25,000 data points. Change the project size. NOTE: Only change the project size if your hardware meets the requirements of a larger project.

After the import, system load will be [more than 90%].

The project size of system data points is nearly reached (current system size + the imported size).

Change the project size to import additional data points.

Trendlog Reference not Correct or Empty

An entry displays in the event list if a trendlog object has an invalid address. The message can be suppressed by:

  • Setting the Logging enabled and Enable logging properties to Inactive.
  • Clear the BACnet reference.

The workflow is described in Section Assigning or Replacing a New Trendlog Reference.

Some Properties are Empty or Display as 0

In Desigo CC V5.0, the updated projects do not display all properties with values (in the Compatibility list, see the section Visibility of Object Properties changed in the standard library V5.0). Reimport the data to update the values.

  1. In System Browser, select Management View.
  2. Select Project > Subsystem Networks > [Network name] > Hardware > [Automation station name].
  3. In Extended Operation tab, select the Import property.
  4. Click Start.
  5. Repeat procedure for each automation station.
  1. The updated values are displayed with the new properties.
