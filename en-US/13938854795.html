Customizing SiPass Import Rules

You can customize the SiPass import rules, for example to do the following:

  • Modify the default Discipline/Subdiscipline/Type/Subtype of the imported objects and define a customized rule to apply to the subsequent SiPass object import.
  • Modify the default import behavior that disregards SiPass objects that are set as disabled in the imported configuration. You may want to import such objects anyway, so that they can be enabled and used at a later date. Note however that this may affect the license requirement of the project.

Import Rules are part of the system Libraries that are only accessible to authorized specialists.

Prerequisites

  • You are authorized and trained to customize the system libraries for a specific level (Country, Region, or Project).
    NOTE: The customization level is set on the System Settings node from the Extended Operation tab.
  • System Manager is in Engineering mode.
  • In System Browser, select Management View.

Create a Customized Import Rules Node

Skip this step if the customized SiPass library structure already exists.

  1. Select System Settings > Libraries > L1 Headquarter > Access > SiPass Integrated > Import Rules.
  2. In the Import Rules tab, the SiPass Import Rules display in read-only mode.
  3. Click Customize , and then click OK.
  4. The customized node is available in System Settings > Libraries > L-customization level > Access > SiPass Integrated > Import Rules.

Edit the Customized Import Rules

  1. Select the customized import rules node. For example System Settings > Libraries > L4-Project > Access > SiPass Integrated > Import Rules.
  2. In the Import Rules tab, open the Instance Attributes Mapper expander.
  3. Do one or more of the following to edit the rules for importing any Object Type:
    NOTE: Do not modify the Default Function Name field.
    • Modify the discipline / subdiscipline / type / subtype. Click in the corresponding column and set the desired new value. For more information see Default Values for Discipline, Subdiscipline, Type, Subtype below.
    • Modify the Skip If Disabled option (True by default) for the object types where this option is applicable. Selecting False will result in importing all objects even if they are disabled.
  4. Click Save .
  5. Any change to the rules will affect all objects of the modified types involved in subsequent imports.
  6. Import the SiPass configuration again to apply the new settings.

Default values for Discipline, Subdiscipline, Type, Subtype

For each object type, default values are indicated by “…” in the table and are set to:

- Discipline = Security
- Subdiscipline = Access Control
- Type = the default for each object type (such as, Area, Device, Door or Input)
- Subtype = Unassigned

For more information about the default setting of an Object Type, see the corresponding model in System Settings > Libraries > L1 Headquarter > Access > Common > Object Model.
