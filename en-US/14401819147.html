Aguilera Compatibility and System Limits

Compatibility

  • Aguilera AE/SA-C2, AE/SA-C8, AE/SA-C23H and AE/SA-C83H algorithmic central units (Communication Interface Firmware Version: AGE48 v2.0)
  • Desigo CC V3.0 or higher

Size Limits

Since no specific limits are identified for this integration, refer to Desigo CC system limits defined in the System Description document (A6V10415500).

Other Limitations

  • Registers defined in the import rules are valid only if no register customization was configured on the communication interface.
  • This integration covers only the Multiple Status mode of the central unit. Any customization made on the communication interface by changing the Multiple Status mode to the Individual Status mode might result in this integration not to work properly.
  • Due to a limitation in the Desigo CC Modbus Integration, the event commands (Silence, Ack, and Reset) are available for the panel object only. They cannot be replicated in the Operation tab for the zone elements.
  • When a zone event occurs, the Ack and Reset commands become available in Event List with the following limitations for the current version:
    • The Ack and Reset commands are both available at the same time. They remain available also when the acknowledge command has been successfully executed.
    • The event blinks constantly, also after the Ack command has been successfully executed.
    • In order for the Ack and Reset commands to work in Event List, a specific configuration step in the Desigo CC station is required. For instructions, see Configuring Aguilera AE/SA Cx Event Commands Macros.
  • When a zone is in test and an Alarm\PreAlarm event is generated on that zone, both the Test and Alarm\PreAlarm events display on the Desigo CC station, as if a real alarm is pending (and not a test alarm).
