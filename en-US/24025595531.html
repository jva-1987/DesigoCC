OPC Integration

Create an OPC Driver

  1. Select one of the following options, based on where you intend to add the OPC driver:
    • Project > Management Station > Servers > Main Server > Drivers
    • Project > Management Station > FEPs > [FEP station] > Drivers
  2. The Object Configurator is displayed.
  3. Click New object and select OPC Client Driver.
  4. In the New object dialog box, enter the description.
  5. Click OK.
  1. The OPC driver object is displayed in System Browser.

Start an OPC Driver

  1. Select the created driver.
  2. Click the Extended Operation tab.
  3. Click, next to the property Manager status, the option Start.
  4. The status changes to Start successful.

Create OPC Network

  1. Select Project > Field Networks
  2. Click New object and select OPC network.
  3. In the New object dialog box, enter the name and description.
    NOTE: The description must be unique.
  4. Click OK.
  5. The new field network is available in the System Browser.
  6. Click the OPC tab and open the Network Settings expander.
  7. Select the OPC driver in the Selected Driver drop-down list.
  8. Click Save .

Create the Logical View

  1. The Logical View has not yet been created.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Views.
  3. Select the Views tab.
  4. Enter the data in the following fields:
    a. Display name: Logical view
    b. Display description: Logical view
    c. Base object name: Logical
    d. Base object description: Logical
    e. Type: Select logical
    f. Select distribution: Select check box
    g. Supplemental information: (Optional) Enter descriptive text
  5. Click Save .

Import the Project Data Files

  1. One or more valid database files are available locally or by browsing for the reachable network file.
  1. Select Project > Management station > Servers > Main Server > Drivers.
  2. In the Extended Operation tab, select the Transaction Mode property.
    a. In the drop-down list, select Simple.
    b. Click Apply.
    NOTE: The Simple option reduces import time, since not all import transactions are written to the project log database. Following engineering, the project must be reset to Logging.
  3. Select Project > Subsystem Networks > [Network name] and click the Import tab.
  4. The Import page displays.
  5. Open the Hierarchies Mapping expander.
  6. In System Browser, select logical view.
  7. Drag the top node Logical to the Hierarchies expander, to the Logical line in the column Import hierarchy under this node.
  8. Click Open at the top of the page to go to the database files.
  9. In the File type drop-down list, select the data type *.csv.
  10. Select the desired import files. [*].csv.
  11. Click Open.
  12. The preprocessor analyzes the selected file. Click Log Analysis for additional information in the event of a problem.
  13. In the Source column, select .
    NOTE: If Delete unselected items from the views is selected, all automation stations and the linked objects are deleted that are not in the selected import file. The main nodes must be manually deleted in the Logical View and User View.
  14. Click Import.
  15. The import procedure starts with the first file in the Items to Import list. The State can be: Completed, Failed, In progress (%), Partially completed [numbers of failed instances].
    • A message is displayed at the end of the import process that summarizes the results in a detailed log. Click Import log.
      NOTE: The import is not executed if the number of data points exceeds system limits. Check the Number of data points entered in project size.
  16. Select Project > Management station > Servers > Main Server > Drivers.
  17. In the Extended Operation tab, select the Transaction Mode property.
    a. In the drop-down list, select Logging.
    b. Click Apply.

NOTE:
You must reimport if you change the driver configuration after an import.
