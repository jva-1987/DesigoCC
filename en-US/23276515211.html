Populate the Bulk User CSV File

  1. You have reviewed the criteria for each field in following topic: Bulk User Import.
  1. Navigate to \GMSMainProject\profiles\UserDataTemplate\DCC_UserData_Import_Template.csv
  2. Open the CSV file in Excel or the text editor of your choice.
  3. Enter the information for the mandatory field:
    • UserName – A unique user’s name.
      NOTE 1: It is recommended that when you create a new user, the user name should not have a blank space.
      NOTE 2: If duplicates exist, either in the spreadsheet or the user already exists in Desigo CC, then the last entry in the spreadsheet is imported and any existing user information is overwritten. Additionally, roles (see below) from any duplicates are merged with the imported user, resulting in multiple roles for that user.
  4. Enter the information in the remaining optional fields, as applicable:
    • UserType – One of the following user types as presented here: DesigoCC, SoftwareAccount, or OpenID. NOTE: If the field is left blank, the default is: DesigoCC.
    • Description – Description of the user.
    • FullName – An expanded user description.
    • Profile and Flex Client Profiles – The client and Flex profiles applied to the user.
    • Role – The role or user group the user is assigned to.
    • GlobalUser – (Distributed systems and DesigoCC /OpenID users only). Type Yes, if this is a Global user; No if local user.
    • Language – Language code applied for the user. The language must be in the correct format, for example: it-IT and installed in the project. Refer to the System Management Console (SMC) for the project’s installed languages.
    • . Password – (DesigoCC UserType only). The user password.
  5. (Optional) To add multiple roles for a particular user (UserName), add a new line for each role. Repeat as needed.
    TIP: Copy and paste the original username row to keep all the information in the other fields intact and only change the information in the Role field.
  6. Repeat for each user.
  7. Save the CSV file.
