Import BTL Library to an Existing Project

Scenario: You want to replace the Desigo CC standard library with the BTL library on an existing project. The automation station data was previously imported to the project.

 

Reference: General information on Desigo PX automation stations is available at Desigo PX.

 

Flowchart:

 

 

Prerequisites:

  • System Manager is in Engineering mode.
  • System Browser is in Management View.

To speed configuration, make sure the Desigo CC Transaction Mode is set to Simple or the Automatic Switch of Transaction Mode is set to True.
The automatic switch setting is recommended. Without it, to re-enable the logs, you need to set the Transaction Mode to Logging at the end of the configuration procedure.

 

Steps:

1 – (Optional) Check the Number of Data Points

  1. Select Project > Management System > Server > Server > Licenses.
  2. In the sbt_gms_add_ba_dp line, check the number of data points.

 

2 – Import Extended BTL Library

  1. Select Project > System Settings > Libraries.
  2. Select the Library Configurator tab.
  3. Available libraries are displayed.
  4. Click Import.
  5. Open [Installation Drive]: > [Installation Folder] > GMSMainProject > Libraries > Exports > _alternatives.
  6. Select file BA_Device_PX_HQ_2_BTL.gms and click Open.
  7. The file is selected.
  8. Click Yes.
  9. The button is disabled during import.

 

3 – Importing Localized Texts for the Extended Library

  1. Select Project.
  2. Select the Localization tab.
  3. Open the Import expander and click Browse.
  4. Open [Installation Drive]: > [Installation Folder] > GMSMainProject > Libraries > Exports > _alternatives > [Language].
  5. Select the BA_Device_PX_HQ_2_BTL.xml file and click Open.
  6. Click Import.
  7. The import status displays in the Status column.
  8. The import is concluded when the Import window displays.
  9. Click OK.
  1. The localized library texts are imported.

 

4 – Import Data

Even if you deleted the network, you can Start with the Desigo PX network wizard.

  1. Select Project > Management station > Servers > Main Server > Drivers.
  2. In the Extended Operation tab, select the Transaction mode property.
    a. Select Simple from the drop-down list.
    b. Click Apply.
    NOTE: The Simple option reduces import time, since not all import transactions are written to the project log database. Following engineering, the project must be reset to Logging.
  3. Select Project > Subsystem Networks > [Network name] and click the Import tab.
  4. The Import page displays.
  5. Click Open at the top of the page to go to the database file(s).
  6. In the Files of type drop-down list, select the file type exp (one automation station) or zip (includes 1 to n automation stations).
  7. Select the desired import files.
    • Desigo PX files (exp)
    • Desigo PX files (zip)
    • BACnet Router files
    • Web Gateway files
  8. Click Open.
  9. The preprocessor analyzes the selected file. Click Log Analysis for additional information in the event of a problem.
  10. In the Source list, click .
    NOTE: If Delete unselected items from the views is selected, all automation stations and the associated objects that are not in the selected import file are deleted. The main nodes must be manually deleted in Views for Logical View and User View.
  11. Click Import.
  12. The import procedure starts with the first file in the Items to Import list. The State can be: Completed, Failed, In progress (%), Partially completed, [numbers of failed instances].
  13. A message is displayed at the end of the import process that summarizes the results in a detailed log. Click Import log; NOTE: The import is not executed if the number of data points exceeds system limits. Check the Number of data points entered in project size.
  14. Select Project > Management station > Servers > Main Server > Drivers.
  15. In the Extended Operation tab, select the Transaction mode property.
    a. Select Logging from the drop-down list.
    b. Click Apply.
