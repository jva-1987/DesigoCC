Configure the DF8000 I/O via COM Port

You can configure DF8000 I/O modules connected to a remote DF8000 CPU unit that communicates to the NK823x unit over a serial line.
For background information, see DF8000 Reference.

Add the DF8000 with a DF8003 CPU

  1. In System Browser, select Project > Field Networks > [BACnet network].
  2. Select the Object Configurator tab.
  3. Click New and select New GMS_DF8000_Serial.
  4. In the New Object dialog box:
    a. (Optional) Edit the precompiled name and description (DF8000 System via COM Port).
    b. Click OK.
  5. The new DF8000 System via COM Port object is added and automatically selected in System Browser.
  6. Select the DF8000 tab.
  7. In the Configuration expander, enter the Instance number (the BACnet Device Identifier).
  8. Click Save to enable the remaining fields.
  9. In the Configuration expander:
    • Select the communication Baud Rate
    • Select the Line Type (for COM 1 and COM2: RS232 or RS485)
    • Select Support CF9003 if you want to connect a CF9003 unit
      NOTE: 19200 bps Baud Rate is not allowed for CF9003
  10. Click Save .
  11. A new DF80003 CPU, with address set to 0, is added to System Browser, and to the Units expander of DF8000 System via COM Port.

 

Add Input / Output Modules to the DF8003 CPU

Repeat this procedure for all the input / output modules connected to the DF8003 CPU.
NOTE: You cannot mix supervised modules (DF8045 and DF8046) with On/Off modules (DF8020 and DF8040) on the same CPU. Instead, different I/O units can coexist on the same RS485 bus until the maximum addressing limit is reached. See Valid DF8000 I/O Configurations on I2C Bus.

  1. In System Browser, select Project > Field Networks > [BACnet network] > [DF8000 system via COM Port] > [DF8003 CPU].
  2. Select the DF8000 tab.
  3. Click New and select the type of module you want to add:
    • DF8040 Digital Input Module
    • DF8020 Relay Output Module
    • DF8045 Balanced NC Input Module
    • DF8046 Balanced NO Input Module
  4. In the New Object dialog box:
    a. (Optional) Edit the precompiled name and description.
    b. Click OK.
  5. The new module is added to System Browser, and to the Modules expander of the DF8003 CPU.
  6. In the Modules expander, select the newly added module.
  7. Enter an Address (1-6) that matches the setting on the hardware module. Be aware that addressing errors are not notified as fault events.
  8. Click Save .

 

(Optional) Add Other DF8003 CPUs with I/O Module

Do this procedure if the hardware configuration includes additional DF8003 CPUs.

  1. In System Browser, select Project > Field Networks > [BACnet network] > [DF8000 system via COM Port].
  2. In the DF8000 tab, click New .
  3. In the New Object dialog box:
    a. (Optional) Edit the precompiled name and description.
    b. Click OK.
  4. The new DF80003 CPU is added to System Browser, and to the Units expander of DF8000 System via COM Port.
  5. In the Units expander, select the newly created DF8003 CPU.
  6. Enter an Address (0-15) that does not conflict with those already used for other DF8000 CPU units. See Valid DF8000 I/O Configurations on I2C Bus.
  7. Click Save .
  8. Configure the I/O modules connected to this DF8003 CPU. See Add I/O Modules to the DF8003 CPU, above.

 

Connect the DF8000 to the NK823x COM Port

  1. In System Browser, depending on the NK823x serial line connection, select one of the following:
    Project > Field Networks > [BACnet Network] > [NK823x] > [com port]
    Project > Field Networks > [BACnet Network] > [NK823x] > [Expansion Board] > [com port]    .
  2. Select the NK8000 tab.
  3. In the Configuration expander, from the Control unit drop-down list, select the DF8000 System via COM Port configured above.
  4. Click Save .
