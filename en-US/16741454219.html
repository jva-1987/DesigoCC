Controlling Incidents

This section describes how to suspend, resume, cancel, and close an incident. In addition, this section provides information on adding and launching a new notification under an existing incident. Save an existing incident under a new name using Save As.

 

Suspend an Active Incident

  1. The incident to be suspended is in Active state.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. To search the active incidents, click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In Status drop-down menu, select the Active check box.
  7. Click Search.
  8. Select the active incident to be suspended.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Status expander.
  11. Click Suspend.
  12. Click OK.
  1. The incident is suspended.

 

Resume a Suspended Incident

  1. The incident to be resumed is in Suspended state.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. To search the suspended incident, click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down menu, select the Suspended check box, and click Search.
  7. All suspended incidents display.
    NOTE: Down the search further using some other search options available in the Advanced Search Options dialog box.
  8. Select the Suspended incident to be resumed.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Status expander.
  11. Click Resume.
  12. Click OK.
  1. The Suspended incident is resumed.

 

Perform an All Clear for an Active or Suspended Incident

  1. The incident to be cleared is in Active or Suspended state and the associated incident template contains an all clear launch group to perform the all clear operation.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. To search the active or suspended incident click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down menu, select the Active or Suspended or both check boxes, and click Search.
  7. All active and suspended incidents display.
  8. Select the incident for all clear.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Status expander.
  11. Click All Clear.
  12. Click OK.
  1. The incident state is changed to All Clear.

 

Cancel an Active or Suspended Incident

  1. The incident to be canceled is in Active or Suspended state.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. To search the active or suspended incidents, click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down menu, select the Active or Suspended check box, and click Search.
  7. All active and suspended incidents display.
    NOTE: Down the search further using some other search options available in Advanced Search Options dialog box.
  8. Select the active or suspended incident to be canceled.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Status expander.
  11. Click Cancel.
  12. Click Yes.
  1. The active or suspended incident is canceled.

 

Close an Incident

  1. The incident to be closed is either in Canceled or Completed state.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. Search the canceled or completed incidents using either of the following search options:
    • Enter the incident name in Search field, and click .
    • Click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down list, select the Canceled and Completed check box, and click Search.
  7. All the canceled and completed incidents display.
  8. Select the incident to be closed.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Status expander.
  11. Click Close.
  12. A confirmation message displays.
  13. Click Yes.
  1. The incident is closed.

 

Add an Additional Notification

  1. The incident to which an additional notification is to be added is initiated and in Active state.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. To search the required incidents, click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down menu, select the Active check box, and click Search.
  7. All the incidents in Active state display.
    NOTE: Down the search further using some of the other search options available in the Advanced Search Options dialog box.
  8. Select the incident to which an additional notification is to be added.
  9. The Incident Status and Incident Structure sections display.
  10. To add an additional notification to the incident, do one of the following:
    • Select and drag a notification from the System Browser to the required launch group. Select the Incident Structure expander.
    • In the Incident Structure expander, select the launch group and click Add.
  11. The Add Notification Template to Incident Template dialog box displays.
  12. Select a notification template and click Add.
  1. The additional notification is added to the incident.

 

Launch an Additional Notification

  1. The incident to which an additional notification is to be added is initiated and in Active state.
  2. The additional notification is added to the incident.
  3. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated incidents displays.
  4. To search the required incidents, click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down menu, select the Active check box, and click Search.
  7. All the Active incidents display.
  8. Select the incident to which an additional notification is to be added.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Structure expander.
  11. Click Launch.
  12. The Launch Additional Notifications for Incidents dialog box displays.
  13. Click Launch.
  1. The additional notification is launched.

 

Save an Existing Incident with a New Name

  1. The incident to which an additional notification is to be added is initiated and in Active state.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Browse Incidents.
  3. A list of all the initiated Incidents displays.
  4. To search the required incidents, click Filter .
  5. The Advanced Search Options dialog box displays.
  6. In the Status drop-down menu, select the Active check box, and click Search.
  7. All incidents in Active state display.
  8. Select the incident to which an additional notification is to be added.
  9. The Incident Status and Incident Structure sections display.
  10. Open the Incident Structure expander.
  11. Click Save As.
    NOTE:     Save the existing incident with or without updating.
  1. The existing incident is saved under the new name.
