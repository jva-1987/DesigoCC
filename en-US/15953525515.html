BACnet Fire 3rd Party TXT Files

Mapping of the Hierarchical Structure in System Browser

Multiple views of system objects are possible in System Browser depending on the integrated field networks:

  • Project data is only displayed in Management View and no additional configuration is provided. The information is taken from the Object Name column to create the objects in Management View.
  • In addition, a logical view or user view can be defined by creating with a text editor the corresponding configuration file (respectively, Hierarchy-Logical.txt or Hierarchy-User.txt).

Format of the Logical and User Views TXT Files

!encoding!utf-8
#separators-1#;
#separators-2#,;
:column-name:keyname
;Family;BACNET_Fire
+EOType-Aggregator+EO_BA_AGG_1
?EOType-Folder?EO_BA_FDR_1
,',1,100
,',1,100
,',1,100
,',1,100
,',1,100
,',1,100

Each TXT file comprises the following sections:

  • EDE file design
  • Hierarchy mapping design

 

Any character can be used as the definition separator, but it must not occur in the corresponding line.
