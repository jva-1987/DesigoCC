Integrating Video Surveillance with Hanwha Samsung SSM Provider

To use the Hanwha Samsung SSM provider for video surveillance in Desigo CC, you must follow the main integration workflow, Integrating Video Surveillance, located in the Video engineering step-by-step section, with the variants specified here.

Main Configuration Workflow

  1. In Set Up Video Devices and the VMS, follow the instructions as given.
    To install the VMS server, see Installing the Hanwha Samsung SSM Provider Software.
  2. In Configure Project Settings for Video, follow the instructions as given completing the information for the Hanhwa Samsung VMS provider. See Setting the Samsung VMS Provider.
  3. In Set up User Accounts and Networking for Video, do not perform the instructions given, which are for the Milestone/Siveillance VMS. Instead, perform the following: Configuring Video API for Hanwha Samsung SSM.
  4. Complete Acquire the Video Configuration from the VMS as given.

 

If Windows App client is used, the provider must be installed on each Desigo CC client.

 

In complex architectures with multiple connections to the SSM (that is, multiple Desigo CC clients that connect to the Desigo CC server), it is necessary to configure an individual user account for each Desigo CC client.

 

Remember to update SSM user account passwords due to expire. If those passwords expire, the video system will not work in Desigo CC.
