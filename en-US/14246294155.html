Additional Procedures for Schindler Elevators

Stop Schindler OPC Driver

  1. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > Management station > Servers > Main Server > Drivers > [Drivers].
  3. In the Extended Operation tab, next to the Manager status, click Stop.
  1. A confirmation message appears.

Delete the Schindler Network

  1. System Manager is in Engineering mode.
  2. The Schindler OPC network is on the Field Networks list.
  1. In System Browser, select Management View.
  2. Select Project > Field Networks > [Network name].
  3. Click the OPC tab.
  4. Click Delete Object .
  5. A confirmation message displays.
  6. Click Yes.
  1. The System Manager status bar displays Node deleted successfully.

NOTE:
The network structure is deleted in the Management View only. It still remains in the Logical View and the User View. In those views, you have to delete the structure manually. Select the desired view and then the Views tab. Click Remove to delete the structure tree and use the purge function to delete all unused objects.

Purging Unused Objects

  1. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select the desired view and select the top object.
  3. Click the Views tab.
  4. In Views, select the top object.
  5. Click Purge Aggregators and then Yes.
  6. All objects without a child object are deleted in this view.
  7. Repeat the Steps 2 through 5 for all views.

Defining Travel Heights for Escalators

  1. System Manager is in Operation mode.
  2. The elevators and escalators are online.
  1. In System Browser, select logical view.
  2. Select Logical > [Subsystem name] > [Building name]
  3. The elevators and escalators are displayed in the Standard tab.
  4. Proceed as follows to select the escalator:
    • Click the escalator title on the graphic.
      or
    • Select Logical > [Subsystem name] > [Building name] > [Group x] > [Escalator x].
  5. In the Extended Operation tab, select the Floor start property.
    a. In the Value drop-down list, select the floor.
    b. Click Change.
  6. In the Extended Operation tab, select the Travel height property.
    a. Enter the difference between the floors (a maximum of 3 floors) in the Value field.
    b. Click Change.

Required Customized Modifications

A Schindler project must always be customized:

  • Assign passenger alarm to an alarm category based on the applied alarm profile
  • Add customized text group extensions
  • Color assignments in the graphics symbol for the alarm display
  1. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L1 Headquarters > Transport > Device > Schindler.
  3. Click Customize .
  4. The customized folder for text groups and object model are created.
  5. Proceed as follows:
    a. In the object model, change the alarm class for the passenger alarm.
    b. Expand the text groups
    c. Expand the alarm color for the Graphics symbol.

Assigning a Passenger Alarm to the Alarm Profile

  1. The customized folder for text groups and object model are created.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L1 Headquarters > Transport > Device > Schindler > Object models > Car.
  3. Click Customize .
  4. The customized object model is created.
  5. Select Project > System Settings > Libraries > L4 Project > Transport > Device > Schindler > Object models > Car.
  6. Select the Models & Functions tab.
  7. In the Properties expander, select the PassengerAlarm property.
  8. In the Alarm configuration expander, select the column Alarm class.
  9. In the drop-down list on the second line, select option LifeSafetyManual.
    NOTE: The assignment for Alarm class to the alarm category must match the alarm profile.
  10. Click Save .

Checking Event Response Offline

This check does not replace the physical test of the passenger alarm in an elevator. It is merely used to check the assignment to the event profile on the Event bar.

  1. Select Start > All Programs > [Company name] > Desigo CC > Tools > PARA.
  2. Enter the system password.
    NOTE: The system password is defined on the System Management Console.
  3. Click OK.
  4. Select the elevator object GMS_OPC_[Network name]_Car_01.
  5. Select the property PassengerAlarm for an elevator.
  6. In the Values group field, select property Original value.
  7. Enter 1 and click Apply.
  8. The event is displayed in the Alarm Summary Bar. The alarm class must be changed if the alarm is not displayed.
  9. Check the passenger alarm for each elevator directly in the car.
  10. Generate a test report on the triggered alarms and hand it over to the principal.

Modifying Alarm Color on the Graphic Symbol

Scenario: The graphics symbol for assigning the alarm color must be supplemented using the customized extension by the customer.

  1. The customized folder for text groups and object model are created.
  1. Select Project > System Settings > Libraries > L1 Headquarters > Transport > Common > Graphic Symbols.
  2. The Graphics Editor displays in the Default tab.
  3. Select symbol DYN_ALL_OperatingMode_ListOfColor_None_None_001.
  4. Click Customize .
  5. The customized symbol is created.
  6. Select Project > System Settings > Libraries > L4 Project > Transport > Common > Graphic Symbols.
  7. Select the Default tab.
  8. Select symbol DYN_ALL_OperatingMode_ListOfColor_None_None_001.
  9. Click Edit .
  10. Select the View tab.
  11. Select the Evaluation Editor check box.
  12. In the Evaluations Editor expander, select the column Condition.
  13. Enter the corresponding value (from the text group) for the alarm color using a separator in line 1,2 or 3.
  14. Click Save .

Logging Data Values

Scenario: You want to permanently log data point values on all elevators to display and evaluate at a later date as needed. The change is made in the object model and impacts all elevators equally.

  1. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L1 Headquarters > Transport > Device > Schindler > Object models > [Elevator].
  3. Click Customize .
  4. The customized object model is created.
  5. Select Project > System Settings > Libraries > L4 Project > Transport > Device > Schindler > Object models > [Elevator].
  6. Click the Models & Functions tab.
  7. In the Properties expander, select the property in question (for example CommandOperationMode.Mode) and select the VL check box.
    Note: The value is entered in the database for each change of state.
  8. (Optional) Select an Archive group.
  9. Click Save .
