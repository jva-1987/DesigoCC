Notification Profiles

You create notification profiles when you want the system to issue a warning message when a meter is nearing its maximum limit or is displaying incorrect data.

You can create a profile for any of the following cases:

  • Meter rolls over to a new value – In this case, the system issues a warning when the value of the meter reaches the specified warning limit. The warning limit is specified as a percentage of the maximum meter reading that is configured in the Managed Meters application. To handle a meter rollover, you must select the Meter Roll Over check box in the Notification Profile expander. You must also specify the warning limit (in percent). For example, if you specify the maximum meter reading of the managed meter as 100000, and set the warning limit to 85%, then the system issues a warning when the meter reading is 85000.
  • Meter reading shows a value that is less than the previous value – In this case, the system issues a warning when the meter value is less than its previous value. For example, if the previous value of the meter was 1000, and the meter rolls down to 998, a warning is issued. In order to issue the warning for a negative meter reading, you must select the Meter Reading Decrease check box in the Notification Profile expander.
  • Meter reading displays zero – In this case, the system issues a warning when the meter reading displays 0. For example, if the previous value was 998, and the meter jumps down to 0, then a warning is issued. To set this warning, you must select the Zero Meter Reading check box in the Notification Profile expander.

Default Notification Profile - Whenever you create a new managed meter from the Managed Meters application, a default notification profile (HQProfile) is assigned to this meter. You can change the default to a different profile and save the managed meter configuration. The default notification profile cannot be saved or deleted. However, you can create a new profile from the default profile using the SaveAs function.
