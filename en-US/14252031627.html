Troubleshooting Schindler elevators

Handling Import Error Conditions

Message

Situation

Do the following…

Invalid CSV File Format

The selected file is an invalid data file, not created with SLV_OPCExplorer.exe.

Create a new CSV export file with SLV_OPCExplorer.exe and attempt to import it.

Import failed – Unable to create a new data point.
Abort, Continue, Continue All

The import did not recognize one of the objects for import.

If you do not know the exact reason for the missing object type, click Abort to stop the import procedure.

Alternatively, you can choose to Continue and skip the missing object or click Continue All to ignore any further recoverable errors.

In any case, try creating a new XML file and re-importing it. If the problem persists, contact Technical Support.

After the import, the system load limit is exceeded.

The import function recognized more points than defined on the project. The import is cancelled after confirming the message with OK.

The project is set to Small by default and supports 25,000 data points. Change the project size. NOTE: Only change the project size if your hardware meets the requirements of a larger project.

After the import, system load will be [more than 90%].

The project size of system data points is nearly reached (current system size + the imported size).

Change the project size to import additional data points.

The SLV_OPCExplorer Tool Does not Start

Situation

Possible Causes

Do the following…

The SLV_OPCExplorer Export Tool does not start.

An older version of the OPC Core Components is installed on the computer.

From the Desigo CC distribution media, open folder ..\GMS\Prerequisites\OPC Core Components and reinstall the OPC Core Components Redistributable (x64).msi.

No Communication with the OPC Server

Situation

Possible Causes

Do the following…

No communication is available.

The Desigo CC OPC driver has not started.

Start the Desigo CC OPC driver.

No Communication with the Schindler OPC Server

Softing dataFEED communications has not started. is displayed on the taskbar.

Right-click and select Start. The state changes to .

NOTE: The state must be the same on the Schindler server as on the Desigo CC computer.

No communication, although state is displayed on both computers.

Conflict with Desigo CC drivers and Softing dataFEED drivers.

1. Stop the Desigo CC OPC driver.

2. Stop Softing dataFEED on both computers.

3. Restart all drivers.

No online data is displayed.

The import file is not up-to-date.

Create a new export file and import the new data.
