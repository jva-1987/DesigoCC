Start Data Import with Scan Devices

You selected Scan device in step 6.

 

(Optional) Search Device ID with the BACnet Object Browser

  1. Open the Windows Start menu and select All Programs > [Vendor Name] > Desigo CC> Tools > BACnet Object Browser.
  2. The BACnet Object Browser scans the network for BACnet devices.
    Note: If multiple networks have been defined, they are not displayed using the defined network names. The designation in the BACnet Object Browser is System1_1, System1_2, System1_3, and so on.
  3. Select a device from the system tree in the left pane.
  4. The right pane is updated with all associated information on the device, sorted by property name and value column.
  5. For subsequent AutoDiscovery in tab Network devices (application of a filter), note the following data:
    • vendor-identifier
    • object-identifier
  1. The device data for the online import into Desigo CC are established.

No communication is available

Possible cause

Perform the following steps ...

Management station is on the same network:

  • No foreign device defined
  • IP Address does not reference the BBMD device on the network.
  • UDP port not correctly defined.
  1. Select Management View > Project > Management System > Server > Drivers > [driver name].
  2. Select the BACnet tab and open the BT BACnet Stack Config expander.
  3. Check the settings for Port Properties > Foreign-Device.
    The IP address must reference the BBMD device of the network.

BACnet driver not working.

  1. Select Management > Project > Management System > Server > Drivers > [driver name].
  2. Click the Extended Operation tab.
  3. Select Manager Status and click Start.

The firewall does not allow the BACnet driver.

  1. Add GMS BACnet Driver under Allow communication of Apps through Windows Defender Firewall.

Starting a Network Scan

  1. You have completed the optional Scan by Device ID workflow and the information needed to filter is available.
  1. Select the Network device tab.
  2. Open the Discovery Settings expander.
  3. In the field Scan timeout in seconds, enter the maximum duration of a scan (Default: 60 seconds).
  4. Select Enable device discovery.The Desigo CC project data base is automatically updated if the Desigo CC project data base deviates from the data base ID on the automation station.
  5. Select the filter criterion:
    • Use network filter and enter the Network number filter.
    • Use instance filter and enter the Beginning instance and Ending instance (object-identifier).
    • Use vendor filter and enter a Vendor ID.
    • Use a Specific instance filter and enter the instance number.
  6. Click Save.
  7. Select the Device recognition expander and click Scan. The device scan operates for the period entered in Scan timeout in seconds after the set period if the Cancel button is selected. Cancel prematurely cancels the scan.
  8. Discovered devices are displayed in the Device recognition expander in the Network scan pane.
  9. Select check box Import selection for each automation station for import.
  10. Click Import.
  11. Click Yes.
  1. The progress displays in the Scan status section. This may take a few minutes depending on the amount of data.
  2. Data points are displayed in System Browser below the selected network.
