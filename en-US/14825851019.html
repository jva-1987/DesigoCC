Notifier ID3000 Compatibility and System Limits

Compatibility

  • ID3000, ID3002, ID50, ID60 fire control panels via IntesisBox Communication Interface Firmware Version: 42.0.8
  • Desigo CC V3.0 or higher

Size Limits

Since no specific limits are identified for this integration, refer to Desigo CC system limits defined in the System Description document (A6V10415500).

Other Limitations

  • This integration supports only the Bit Coded mode of the IntesisBox, which allows configuring the wanted value in the Modbus register for each possible state of the panel elements.
    To allow this integration to work properly, avoid any customization on the Communication Interface (such as, changing the default setting of the option States in Modbus Register Bit Coded or disabling it).
  • Due to a limitation in the Desigo CC Modbus integration, the event commands (Ack and Reset) that are available on the panel object can be replicated on the panel node only. They cannot be replicated in the Contextual pane on zone, detector and module elements.
  • When a zone, detector, or module event occurs, the Ack and Reset commands become available in Event List with the following limitations for the current version:
    • The Ack and Reset commands are both available at the same time. They remain available also when the acknowledge command has been successfully executed.
    • The Reset command is available in Event List also for Fault events on zone, detector, or module elements.
    • The event blinks constantly, also after the Ack command has been successfully executed.
    • Symbols in the graphic never blink. This means that the operator cannot determine acked and unacked conditions from the graphics.
    • For the Ack and Reset commands to work in Event List, a specific configuration step is required in Desigo CC. For instructions, see Configuring Notifier ID3000 Event Commands Macros.
