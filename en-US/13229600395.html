Applying an Event Schema to a Project

Event schemas define the mapping of alarms to categories, colors, and sounds. You can change the event schema that is applied to a project. For background information, see Event Schemas.

 

  1. You are trained and authorized to work with system libraries at the target customization level. For example, L4-Project.
  2. System Manager is in Engineering mode.
  3. No alarms are currently active in the system.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L4-Project or L3-Country or L2-Region > Global > Events > Schemas > [event schema].
  3. In the Event Schema tab, click Apply schema to current project .
  4. By default, each event schema uses the text groups of the L1-Headquarter library. If you want to use custom text groups for an event schema, you must first apply the event schema and then customize its text groups.
  5. If the new Event Schema adds new types of events to the project, enable them in the Event Rights expander of the Security node.
    For instructions, see Configuring Event Rights.
  6. To align the Desigo CC client application and the project with this new configuration, do the following:
    a. On each of the Desigo CC installed stations, make sure a Client Profile is applied that is compatible with the new event schema.
    See Set Profile for an Installed Station.
    b. On each of the Desigo CC Flex web stations, make sure a Flex Client Profile is applied that is compatible with the new event schema.
    See Set Profile for Flex Web Station.
    c. Shut down the Desigo CC application on all the installed stations.
    d. Exit the Desigo CC Flex Client application on all the Flex web stations.
    e. Using the SMC (System Management Console), stop and restart the project.
    f. Start up the Desigo CC application on all the stations.
    g. Run the Desigo CC Flex Client application on all the Flex web stations.
  7. (Optional) If you do not want to keep in the database the events were logged with the old event schema, you must purge the History Database.
    See Purge HDB Data Content.
