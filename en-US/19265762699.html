OPC UA Client Connectivity

The OPC UA Client extension allows configuring OPC UA southbound connectivity through the OPC UA adapter. The OPC Unified Architecture (OPC UA) is a communication protocol for industrial automation.

Engineering for OPC UA Connectivity

The adapter software is not part of the Desigo CC installation. After installation, the adapter installer package (MSI) is available in [Installation Drive]:\[Installation Folder]\GMSMainProject\AddSW\OPC_UA_Client. Once you install and start this software service, the adapter is ready to be executed and used in Desigo CC. For instructions, see Installing and Starting the OPC UA Client Adapter.

 

To integrate the OPC UA connectivity, a SORIS driver and a SORIS network (monitored by one SORIS driver) must be created and configured manually or using the network wizard.

The OPC UA data to be then discovered in Desigo CC by the OPC UA adapter can be configured in one of the following ways:

  • Online engineering. In online configuration, you use the OPC UA Client Adapter web client to connect to an OPC UA server and retrieve its online data.
    For instructions, see Performing OPC UA Client Adapter Online Configuration.
  • Offline engineering. In offline configuration, a file is already available and contains data of an OPC UA server with which a connection was previously established. You can adjust and save the updated configuration through the OPC UA Client Adapter web client.
    For instructions, see Performing OPC UA Client Adapter Offline Configuration.

Dependencies

The OPC UA Client extension is configured to depend on the following extension, which will be automatically added, if it is not already installed in the system, when the OPC UA Client installation feature is selected:

  • SORIS Driver (see SORIS)
