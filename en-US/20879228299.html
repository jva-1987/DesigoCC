Hardware Category and System Limits

The Desigo CC software has a Hardware Category parameter that it uses to define system limits (the maximum number of System-Browser-tree objects that a project can have). The Hardware Category can be set to:

  • EP = (embedded computer) maximum 10000 objects
  • MS = (small or medium sized server/FEP) ≤50000 objects
  • LS = (large-sized server maximum) >50000 and ≤150000 objects

Desigo CC then monitors how close a project is operating to those system limits, and

  • Generates an alarm if the maximum permitted number of objects is exceeded.
  • Prevents you from performing imports that would exceed the system limits.
  • Warns you when you are approaching the system limits.

This avoids instability that may arise from overloading the hardware.

By default, the hardware category is set to MS (≤50000 objects), but you may need to adjust this value to EP or LS to match the actual scale (number of field devices and subsystems, and so forth) of the installation you are configuring.

The System Description document (A6V10415500) details the criteria for determining the system dimensioning, and the hardware requirements for each category.

For instructions, see Configuring Hardware Category and System Limits.

Hardware Category and System Limits Properties

In Engineering mode, when you select the Main Server object in System Browser, information about the hardware category and system limits displays in the Extended Operation tab.
The various properties and commands of the Main Server object, which also include those relating to the hardware category and system limits.

If the project only slightly exceeds the maximum system objects for its hardware category, contact Customer Support to increase the System Objects Limit for that category.
However, if the number of objects is significantly above the limit, the project must be adjusted to a higher category, which may also require an upgrade of the computer hardware. Contact Customer Support for assistance.

System Load

Loading of the system with respect to the system limits.

  • If the number of objects is within the system limits and also below the Notification Threshold, the System Load property displays in the Extended Operation tab only, and indicates Normal.
    NOTE: Import operations that would not exceed the Notification Threshold are carried out directly.
  • If the number of objects is within the system limits but above the Notification Threshold, the SystemLoad property displays in the Operation and Extended Operation tabs, and indicates Warning.
    NOTE: In case of an import operation that would exceed the Notification Threshold, a warning message informs you of how close you will come to the limit and lets you choose whether to proceed.
  • If the system limits are exceeded (for example, because of objects auto-discovery), the SystemLoad property displays in the Operation and Extended Operation tabs, and indicates Alarm. Also, an alarm occurs in the system.
    NOTE: In case of an import operation that would exceed the system limits, an error message informs you that the import operation will not start.

Other Properties

  • System Properties: Total number of the properties available in the project.
  • Drivers Properties: Total numberof properties handled by the drivers.
  • Alarm Properties: Total numberof properties that can generate alarms.
  • Text Records: Total numberof string entries in the project database.
  • Value Records: Total number of numeric values entries in the project database.
  • List Records: Total number of arrays (complex data structure entries) in the project database.
