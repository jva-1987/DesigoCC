Web Server (IIS)

Web servers allow a web browser to access the system on the customer's intranet or the Internet. A web server is required to use the Desigo CC Windows App Client or the web services.

The Desigo CC web server is based on Microsoft Internet Information Services (IIS). IIS needs to be installed on each computer serving as the web server. IIS installations on remote computers are supported, such as in demilitarized zone (DMZ).

A web server can be installed on the same hardware running the Desigo CC Server (known as a local web server) or on a separate, dedicated computer (known as a remote web server).

The latter applies for instance, if the customer’s IT department requires using existing web servers, to be installed in a separate controlled environment, or if it is preferred not to use the resources of the system server for IIS.

Alternatively, the Client or FEP stations can also act as a station hosting a web server (IIS).

To work with Windows App Client on a remote web (IIS) hosted on Client/FEP station, it is recommended to first install the IIS before installing the setup type Client or FEP.

During the installation, the Installer, installs the software component Microsoft Application Request Routing (ARR), which is required for working with websites/web applications in the System Management Console (SMC).

By browsing the website/web application URL generated in SMC in the supported browser, such as Microsoft Edge, Chrome, Firefox or Microsoft Internet Explorer 11 and above, a Desigo CC web page displays.
