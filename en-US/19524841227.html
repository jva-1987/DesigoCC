This section describes the effects of a specific Client Profile on the Flex Client user interface.

Notification Center

Function

Description

Details

Buzzer

Cannot edit sound settings.

In the Notification style settings, the feature Play sound for unprocessed events is switched on permanently. The operator can never disable the possibility to play a sound when a new event occurs.

Toast messages in Notification Center

Enable notifications for new events.

In the Notification style settings, the operator can switch on or off the feature Show in Notification Center, which allows showing toast messages for new events in the Notification Center.

Event List

Function

Description

Details

Event filters

Disable automatic removal of applied filters on new events.

Any applied filter criteria are not automatically cleared when a new event occurs.

Event grouping

Disable grouping together events that occur when the same event source repeatedly generates the same alarm condition.

Events belonging to the same source and category display grouped in Event List.

Colored rows

Not Applicable

The event row color does not match the event category color.

Columns

Indicate Event List default columns.

  • Discipline/category (event icon)
  • Event cause
  • State
  • Date/time
  • Timer
  • Source
  • Belongs to

The operator can resize, move, or hide the columns, and restore default columns layout.

The event icon is the only column that the operator cannot resize, move, or hide.

The operator can show or hide additional columns but cannot resize the Commands column.

Sorting criteria

Indicate default event sorting in Event List.

  • State: Unprocessed (to be Ack) events grouped on top, then Ready to be reset events, then events that are Waiting for condition
  • Category: most severe on top, for example, Life Safety before Fault
  • Date/time: most recent event on top

With this sorting, all events to be acknowledged will be grouped at the top. Within the ‘to be acknowledged’ group, events will be grouped by category. Finally, within a group of ‘to be acknowledged’ events belonging to the same category, the most recent ones will display on top.

Disable sorting

Not Applicable.

The operator can change default event sorting.

Disable multiple selection

Not Applicable.

The operator can select multiple events in Event List to handle with one command.

Disable event commands on active event filter

Not Applicable.

The operator can always send event-handling commands, when available.
