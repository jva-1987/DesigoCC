Operating and Testing Emergency Lighting

Emergency lighting is generally switched on and off in groups. Individual tests can be conducted manually for each emergency light.

The test workflow is as follows:

  • Test emergency lighting
  • Create a report
  • Load battery before starting the next test

Prerequisites:

  • The System Browser is in Operating mode.
  • In System Browser, Management View is selected.

 

Switching On and Releasing Group

Emergency lighting can be switched on in advance if the time of the power outage is known.

  1. Select Applications > Logics > Central Functions > [Hierarchy Name] > Emergency Lighting [Category] > CenEmgLgt [Emergency Lighting (CenEmgLgt)].
  2. In the Operation tab, select the Emergency Lighting Control Command property.
  3. Click Manual.
    a. In the Value drop-down list, select an option.
    b. Click Send.
  1. Emergency lighting is switched on at priority 8.

 

Manually Testing Group

  1. Select Applications > Logics > Central Functions > [Hierarchy name] > [Hierarchy 1-n] > [Central Functions for Emergency Lighting] > [Central Function Emergency Lighting for Zone A (CenEmgLgt ZoneA)].
  2. Select the Operation tab.
  3. Displays emergency lighting properties.

Starting Function Test

The function test briefly places the device in a power outage state. The test does not impact battery capacity.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Start function test.
  3. Click Change.
  4. The function test starts .
  5. The function test concludes after about one minute .
  6. The test result must be read from emergency luminaires with a report.

Starting Short Duration Test

The short duration test places the device in power outage state long enough to calculate battery capacity without fully discharging the battery.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Start short duration test.
  3. Click Change.
  4. The short duration test starts .
  5. The short duration test takes about 10 to 60 minutes depending on device type .
  6. The test result must be read from emergency luminaires with a report.

Starting Long Duration Test

The long duration test places the device in power outage state long enough to measure battery capacity.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Start duration test.
  3. Click Change.
  4. The long duration test starts .
  5. The long duration test takes about 1 to 5 hours depending on device type .
  6. The test result must be read from emergency luminaires with a report.

Stopping Active Test

Stops a running function or running continuous test. A cancelled test must be repeated within a defined period as per national standards.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Stop active test.
  3. Click Change.

Overriding Rest Operating Mode

Overrides the active power outage state and switches off emergency lighting. Can only be run for active power failure state. The function relieves batteries if the situation is safe.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Go to rest mode.
  3. Click Change.

Switching on Inhibit Operating Mode

Prevents the device from going to the power outage state. Can only be run as long as power is available. Used for planned power outages.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Go to inhibit mode.
  3. Click Change.

Switching off Inhibit Operating Mode

Removes the Inhibit operating mode and switches the device to normal operation. Can only be run as long as power is available.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Cancel Inhibit mode.
  3. Click Change.

Reset Hours of Operation

Resets hours of operation on emergency lighting. Generally used when periodically changing the luminaires. Hours of operation reset must be done on the individual object if only individual defective luminaires are replaced.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Reset lamp op. hours.
  3. Click Change.
  1. The hours of operating are reset .
  2. The hours of operating are reset .

Enabling Switch-off Pulse

Switchable and dimmable devices that are not controlled by a local switch remain in an emergency state after a function or continuous test. The command sets the emergency state to Off.

  1. Central function Emergency Lighting and the corresponding group are selected.
  1. Select the Emergency Lighting Control Command.
  2. In the Value drop-down list, select option Switch-off pulse.
  3. Click Change.

Creating Test Report

Three predefined reports are available to document conducted tests:

  • Function test
  • Short duration test
  • Long duration test
  1. The Emergency Light reports are imported.
  1. In System Browser, select Application View.
  2. Select Applications > Reports.
  3. Open the Reports tab.
  4. Select State and then one of the following reports:
    • Emergency Light Status Function
    • Emergency Light Status Short Duration
    • Emergency Light Status Long Duration
      Note:       These are the standard reports that cover all emergency luminaires in the building. Create your own Reports.
  5. Click Run .
  6. The test report is generated.
  7. Select Create and view PDF .
  8. Click Print .
  1. The test report is printed.

NOTE 1:
The report displays state information in the language defined for the emergency luminaires.
NOTE 2:
A report of the previous test must be created before selecting another test type for testing.

 

Creating Own Report

The existing reports always acquire the entire building. Create your own reports for each zone that is tested.

Create a report

  1. The Emergency Light reports are imported.
  1. In System Browser, select Application View.
  2. Select Applications > Reports.
  3. Open the Reports tab.
  4. Select State and then one of the following standard reports:
    • Emergency Light Status Function
    • Emergency Light Status Short Duration
    • Emergency Light Status Long Duration
  5. Click Save As .
  6. In the New Object dialog box, enter a name and description.
  7. Click OK.
  1. The new document displays in System Browser.

Defining Report Content

  1. The Emergency Light reports are open.
  1. In System Browser, select Logical view.
  2. Select Logical > [Hierarchy name] > [Hierarchy 1-n] > [Room n] > [Emergency Lighting Zone (LgtEmgZone)].
  3. Drag the object to the report.
  4. Repeat Steps 2 and 3 on all objects covered by this report.

Defining Report Output

  1. The Emergency Light reports are open.
  1. Select the Settings tab.
  2. For Report output click the symbol on the bottom right .
  3. The Report Output Definition opens.
  4. Select Report Format (for example, PDF) and define the destination type (for example, file).
  5. Click Configure Folders.
  6. The Report Output Folders Configuration dialog box opens.
  7. Enter the path and alias name of the target folder.
  8. Click Add.
  9. Click Close.
  10. In the Report Output Definition, click Add.
  11. The defined output is displayed in the List of Folders for Report Output.
  12. Click Close.
  13. Click OK.
  14. Click Save .
  1. The report is configured and is saved as a PDF file when running a reaction.

 

Automatically Testing Group

When running tests, remember that there is no automatic feedback that the test is finished or the battery is fully charged. For this reason it is important to calculate sufficient reserve time to start the test report group 1 and test group 2 *charging time of batteries for group 1).

Creating Automatic Execution

The following description applies to a long duration conducted once a year. The example may need to be modified depending on the function selected and project.

  1. The test report is configured and the storage location defined.
  1. Select Applications > Logics > Reactions.
  2. Open the Reaction Editor tab.
  3. Open the Triggers expander.
  4. Open the Time and Operating Mode expander and click Add.
  5. Define the following entries:
    • From date
      ‒ Enter the create date.
    • To date
      ‒ Select check box Any. No end date is defined.
    • Time
      ‒ Enter, under Set start time the execution time 10:00:00.
      ‒ Clear check box Set end time.
    • Effective days
      ‒ For Recurrence, select Monthly.
      ‒ Selection the options First, Sunday every 12 months.
  6. Open the Output > Action expander.
  7. Select Applications > Logics > Central Functions > [Hierarchy name] > [Hierarchy 1-n] > [Central Functions for Emergency Lighting] > [Central Function Emergency Lighting for Zone A (CenEmgLgt ZoneA)] > Emergency Luminaire (EmgLmnr) > Command Emergency Luminiare (EmgLmnrCtl).
  8. Drag the Emergency Light Object to the Action expander.
  9. Define the following entries for the Emergency Light Object:
    • In the Property property, select Present Value.
    • In the Command column, select option Change [Write].
    • In the Command column, in the field Value, select option Start duration test [3].
    • In the Initial delay column, enter a time in seconds [0 = immediate].
  10. Select Applications > Reports > Status > [Test report].
  11. Drag the Emergency Light Report to the Action > Action expander.
  12. Define the following entries for the Emergency Light report:
    • In the Property property, select Not defined.
    • In the Command column, select option Run.
    • In the Delay column, enter a time in seconds [18000 = 5 hours].
  13. Click Save .
  1. The test report is created once the executed reaction is finished.
