External PDFs Naming Convention

If you want to create fire brigade maps from external pdf files, these pdf files must have the correct name to be correctly associated with the corresponding fire objects. For operating instructions, see Importing External PDFs and Running the Fire Brigade Maps Generator.

The name of the file must be formatted as descripted below:

  • <description> only alphabetic characters, not mandatory.
  • <blank> or <_>, not mandatory.
  • <zone number> only number, 1 or 002, 03.. digits, mandatory.
  • <-> separates zone number from detector number, mandatory only if present detector number.
  • <detector number> only number, 1 or 002, 03.. digits, not mandatory.

Linking the PDF file to a specific detector

If the PDF is related to a specific detector, the name of the file must be formatted as:

<description / no description><blank / _ / no separator><zone number><-><detector number / no detector number >.pdf

Example of correct names of PDF associable to zone 14, detector 1:

Geschw_School_LK 14-1.pdf

Geschw_School_LK14-01.pdf

14-01.pdf

Gruppe_101020008.pdf

Example of wrong names of PDF and skipped by the FBM wizard:

Geschw_27-School_LK 14-1.pdf

Linking the PDF file to a specific zone

If the PDF is related to a specific zone, the name of the file must be formatted as:

<description / no description><blank / _ / no separator><zone number>.pdf

Example of correct names of PDF associable to zone 101020008:

101020008.pdf

AAA 101020008.pdf

AAA_101020008.pdf

101020008.pdf

AAA101020008.pdf

Example of wrong names of PDF and skipped by the FBM wizard:

A222A-101020008.pdf

22AAA.pdf

 

The zone/detector number may not be unique throughout the entire Desigo CC project. It should be possible, during the configuration steps for importing and linking, to define a tree structure from any view as a starting point for performing linking.
