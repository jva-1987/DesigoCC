Disabling Alarm Suppression for an Event Schema

By default, the alarm suppression feature is enabled for each event schemas. Due to some restriction depending on specific countries regulations, alarm suppression can be disabled in the event schemas. In such cases, the alarm suppression indicator will not display on the Summary bar and executing the relevant commands in normal operation will have no effect.

  1. You are trained and authorized to work with system libraries at the target customization level. For example, L4-Project.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L1-Headquarter > Global > Events.
  3. Create a customized Events library as follows:
    a. In the Library Configurator tab, click Customize the entire library to a lower level .
    b. Click OK.
  4. The structure of the selected library is cloned under the allowed customization level (in this example, L4-Project > Global > Events).
  5. Add a copy of an existing event schema (for example, EN) to the customized Events library as follows:
    a. In System Browser, select Project > System Settings > Libraries > L1-Headquarter > Global > Events > Schemas > [event schema].
    b. In the Event Schema tab, click Save As.
    c. In the Save Object As dialog box, navigate to and select the schemas block of the library where you want to add a copy of this event schema. In this example, Project > System Settings > Libraries > L4-Project > Global > Events > Schemas. Then enter a name and description, and click OK.
  6. The event schema is added to the library.
  7. Select L4-Project > Global > Events > Schemas > [event schema].
  8. Open the General Settings expander, and clear the Enable Alarm Suppression check box.
  9. Click Save .
  10. Apply the event schema. See Applying an Event Schema to a Project.
