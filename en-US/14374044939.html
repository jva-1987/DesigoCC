Discovering OPC Servers and OPC Items

You can scan the OPC network for OPC servers based upon network numbers, instance numbers, and vendor identification and then import the selected OPC servers and their OPC items.

  1. System Manager is in Engineering mode.
  2. An OPC driver has been added, an OPC network has been added, and an OPC driver has been assigned to the OPC network.
  3. The computer for the OPC discovery has been set. See Configuring OPC DA Discovery Settings.
  1. In System Browser, select Management View.
  2. Select Project > Field Networks > [OPC network].
  3. In the Discovery tab, open the Device Discovery expander. For background information, see OPC DA Device Discovery.
  4. Click Scan.
  5. As OPC servers are discovered, they display in the Network Scan section.
  6. (Optional) From the Library drop-down list, select the library that contains the custom discovery rules to use for the OPC import.
  7. Once OPC servers have been discovered, select the row that corresponds to the OPC server whose OPC items you want to discover.
  8. If required, click Yes to save the changes.
  9. Click Objects.
  10. The Device Discovery expander refreshes and displays any OPC items found.
  11. In the Import Selection column, select the check box that corresponds to the OPC items to import.
  12. Click Back, to go back to the discovered OPC servers.
  13. If required, click Yes to save the changes.
  14. The Import Selection column of the OPC server is now selected.
  15. (Optional) To check the list of the OPC items that are not supported and consequently will not be imported, click Analysis Log.
  16. The Analysis Log dialog box displays its outcome that you can also save in a .txt file.
  17. You can now proceed with the import of the discovered OPC servers and related OPC items. See Importing OPC Servers and OPC Items.
