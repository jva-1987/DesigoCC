Creating a Reaction to Unblock and Unlock Doors in Case of Life-Safety Events

  1. You want the system to automatically unblock doors is a fire alarm occurs. For background information, see Reactions.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Application View.
  2. Select Applications > Logics > Reactions.
  3. The Reaction Editor tab displays.
  4. (Optional) Click to create a new folder, then enter a Name and Description, for example Life safety reactions.
  5. From the Triggers expander, open Event.
  6. Depending on the UL or ULC event schema you use do one of the following:
    • For UL, click New to display a new instruction (row), then,
      in Category, select = Life Safety
    • For ULC, click New four times to display four new instructions (rows), then set Category as follows:
      - Fire Alarm
      - MNS1
      - MNS2
      - Emergency
  7. From the Output expander, open Action.
  8. In System Browser, select the Manual navigation check box.
  9. Expand Field Networks > [SiPass Network].
  10. From the access control network structure, drag the first door to open to ensure a safe way out.
  11. The Action expander displays a new instruction (row).
  12. In Property, select Block Status.
  13. In Command, select Unblock.
  14. Drag the same door again from System Browser to create a new instruction.
  15. In Property, select Lock Status.
  16. In Command, select Unlock.
  17. Repeat steps 9 to 14 as many times as necessary to open all required doors.
  18. Click Save As .
  19. In the Save Object As dialog box, select the main Reactions folder or any subfolder under it as the destination where you want to save this reaction:
    a. Enter a Name and Description, for example Open doors for fire safety escape.
    b. Click OK.
  20. The new reaction is added to System Browser, and is enabled by default.
  21. In System Browser, clear the Manual navigation check box.
