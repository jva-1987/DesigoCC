Troubleshooting OPC DA Server Invalid State When the Project is Stopped

Typically, when a Desigo CC project is stopped, the Desigo CC OPC DA server stops automatically.

It can, however, happen that third-party OPC clients do not respond to the shutdown request when the project is stopped. Consequently, the Desigo CC OPC DA server remains in invalid state, and is inoperable even after restarting the project. Furthermore, also the connected third-party OPC clients might remain in invalid state.

If your third-party client shows this erroneous behavior, do the following:

  1. Select Project > Management System > Servers > Main Server > [OPC DA server].
  2. In the Extended Operation tab, next to the Dismiss OPC Server On Startup property, click Enable.
  3. Its status becomes True. When the project is restarted, any third-party OPC server instance is terminated.
  4. Before starting the Desigo CC OPC DA server, close and restart any third-party OPC client previously connected.
