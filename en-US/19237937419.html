Modifying the Behavior for Connection Lost

By default, in the Settings expander of the OPC tab, the option Show #COM is set, which enables the capability for OPC group of items and OPC field data points to notify the operators if the connection between Desigo CC and any third-party OPC server is lost or not established.

 

In case of project upgrade, the setting manually configured in the config file is taken.

 

If you do not want to provide a notification for this issue, do the following:

  1. Select Project > Field Networks> OPC network > [third-party OPC server].
  2. The OPC tab displays.
  3. In the Settings expander, set the option Do not show #COM.
  4. Click Save .
  1. The OPC driver is stopped and restarted automatically to apply the changes.
    NOTE: If the option Do not show #COM is set when the connection between Desigo CC and any third-party OPC server is lost or not established, #COM will keep on displaying for the OPC items until the connection is re-established.
