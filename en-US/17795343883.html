Configuration Reports for Floors and Rooms

Create Basis for New Configuration Report

  1. System Manager is in Engineering mode.
  1. In System Browser, select Application View.
  2. Select Applications > Reports.
  3. The Reports tab displays.
  4. Click New and select New report .
  5. Click Properties .
  6. Select the Home tab.
  7. Select in the Text menu and select Blank in the drop-down list box. Drag the selected Blank list box to the empty report definition. You can enter a specific text when executing a report, e.g. Floor 1.
  8. Click Save As .
    a. Select folder and enter a name and description in the dialog box.
    b. Click OK.
  1. An empty report is available for further editing in folder Applications > Reports > [storage location]. You can apply this workflow for any report. Additional information on generating reports is available in Reports.

 

Create Lighting Report

The report displays the assignment of buttons and detectors to lighting groups.

  1. An empty configuration report is available for editing.
  1. In the empty text field, enter a text, for example, Floor 1.
  2. Select Table and, in the drop-down list box, select Lighting. Drag the selected Lighting list box to the empty report definition.
  3. Right-click the report definition and select Selected columns.
    a. Configure your report by deselecting columns or reorganizing or adding supplemental columns.
    b. Click OK.
  4. In System Browser, select Logical View.
  5. Select Logical > [subsystem name] > [desired hierarchy level].
  6. Drag the selected hierarchy levels to the report definition.
  7. Click Save As .
    a. Select the target folder and enter a name and description in the dialog box.
    b. Click OK.
    The saved report configuration can be queried at a later date.
  8. Click Run .
  1. The configuration report for lighting is created.

Create Button Assignment Report

The report displays only button assignments.

  1. An empty configuration report is available for editing.
  1. In the empty text field, enter a text, for example, Floor 1.
  2. Select Table and, in the drop-down list box, select Lighting. Drag the selected Lighting list box to the empty report definition.
  3. Right-click the report definition and select Filter and Condition Filters.
    a. In Available columns, select the option Input Type.
    b. In Condition, select the option =.
    c. In Values, enter the option "Button".
    d. Click Add.
    e. Click OK.
  4. In System Browser, select Logical View.
  5. Select Logical > [subsystem name] > [desired hierarchy level].
  6. Drag the selected hierarchy levels to the report definition.
  7. Click Save As .
    a. Select the target folder and enter a name and description in the dialog box.
    b. Click OK.
    The saved report configuration can be queried at a later date.
  8. Click Run .
  1. The configuration report is created for the button assignment.

 

Create Group Master Report

The report shows the assignments between group master and group members. The information is the same as in the Central Function Viewer.

  1. An empty configuration report is available for editing.
  1. In the empty text field, enter a text, for example, Floor 1.
  2. Select Table and, in the drop-down list box, select Groups. Drag the selected Groups list box to the empty report definition.
  3. In System Browser, select Logical View.
  4. Select Applications > Logic > Central Functions [desired hierarchy level].
  5. Drag the selected hierarchy levels to the report definition.
  6. Click Save As .
    a. Select folder and enter a name and description in the dialog box.
    b. Click OK.
    The saved report configuration can be queried at a later date.
  7. Click Run .
  1. The configuration report is created for the group master report.
