User and User Group Administration

This section provides background information on user administration and user group administration. For related procedures, see the step-by-step section.

In particular, this section discusses setting the display and operation rights for the individual user groups and users. The main criteria for issuing user rights are:

  • Which discipline (Ventilation, safety, and so on) belongs to the work environment.
  • Which duties (Engineering, service, monitoring, and so on) must be executed.
  • Which building or section is the work executed.

Desigo CC permits limiting user rights, for example, users can only see and operate their room.

NOTE:
In Desigo CC some user groups are predefined and can be modified and extended by project.

Logon and Access Rights

Four factors essentially determine logon and access rights in Desigo CC:

  • The system must know the user
  • The user must be assigned a user group
  • The user must the appropriate application rights
  • The user must have the appropriate scope rights

The user can log into Desigo CC if these conditions are met. The user may be able to execute the corresponding Desigo CC functions, depending on the assigned rights.

NOTE:
The user must log in as DefaultAdmin the first time Desigo CC is started. The DefaultAdmin password is set while the System Management Console creates the project.
