Configuring Validation

You can configure validation to protect critical system objects against inadvertent changes. When a validated object is changed or commanded in Desigo CC, the user may be required to confirm the action by entering a comment and/or a password, and the operation is logged. For more information see Validation.

Enable Application Rights for Validation

Users must have application rights to configure an object for Validation profile setting and Object version number.

  1. Select Project > System Settings > Security.
  2. In the Security tab, select the user group to be modified.
  3. Open the Application Rights expander.
  4. Expand Object Configuration and, on the level below it, select Configure Validation.
  5. Click Save .

 

Select the Objects to Validate

  1. You have the Application Rights to define validated objects.
  1. In System Browser, select Logical View.
  2. From the drop-down list, select the display mode (Name, description).
  3. In the Search field, enter the name of the object, for example, setpoint.
  4. All objects matching the search criteria are displayed in System Browser.
  5. Select one or more objects.
  6. The selected objects are highlighted in color and the number of selected objects is displayed in the Operation tab.
  1. You can now set or remove the validation for these objects, as instructed in the following procedures.

 

Set a Validation Profile

  1. In System Browser, select the objects you want to protect with validation.
  2. In the Object Configurator tab, open the Main expander.
  3. In the Validation profile drop-down list, select a validation profile:
    NOTE: A profile can also be assigned to all objects in System Browser: for example, Users.
  4. (Optional) Select the Four Eyes check box. Requires user authentication (password), supervisor authentication (name and password), and a comment.
    NOTE: To confirm a modification, the user must have supervise rights for that confirmation.
  5. In the Predef. comments (Predefined comments) drop-down list, select a text group. The assigned text group displays when a comment must be entered by configuration changes.
  6. NOTE: For more information on creating standard texts, see Add a text group. New text groups must be refreshed by pressing the F5 key.
  7. (Optional) Click Change, and enter a number in the Object version field.
  8. Click Save .
  1. All selected objects are immediately validated, and the symbol is displayed in the Operation tab.
  2. The object version is automatically increased by one when saved, unless another specification was entered manually.
  3. Use the Audit Trail Log report to display the validation information in the Comment column.

 

Remove Validation

  1. In System Browser, select the objects for which you want to disable validation.
  2. In the Object Configurator tab, open the Main expander.
  3. In the Validation profile drop-down list, select Disabled.
  4. Click Save .
  1. The selected objects are no longer validated. The object version remains as is and cannot be reset.

 

Reset Object Version for Customer Acceptance

The object version must be reset to 0 or 1 before handing over a validated object to the customer, regardless of what the customer requests as the base value.

  1. In System Browser, select Logical View.
  2. Click Filter search .
  3. The Filter search dialog box displays.
  4. Open the Other drop-down list.
  5. Select Validation Profile and then the options Enabled, Supervised, Four Eyes and Monitored.
  6. Click the gray background or entry fields to close the dialog box.
  7. Click Search.
  8. The found objects are displayed in the System Browser.
  9. Select the first object.
  10. Press the SHIFT key and select the last object.
  11. All objects are highlighted and the Object Configurator tab is selected.
  12. In the Main expander, select the drop-down list Validation profile and Enabled.
  13. Click Change.
  14. In the Object version field, enter 0 or 1 as the base value.
  15. Click Save .
  16. In the Settings expander, select the drop-down list Validation profile, Supervised, Four Eyes and Monitored.
  17. Click Change.
  18. In the Object version field, enter 0 or 1 as the base value.
  19. Click Save .
  1. All validated objects are reset to the base value.

 

Search for Objects with Four Eyes Validation

You can search for all the objects for which Four Eyes validation is set as follows:

  1. In System Browser, click Filter search .
  2. Open the Other drop-down list.
  3. Select the Four Eyes check box and then the option Enabled.
  4. Click the gray background or entry fields to close the dialog box.
  5. Click Search.
  1. The found objects are displayed in System Browser.
