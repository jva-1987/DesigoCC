Configure a Display Rule Triggered by an Object Selection

To configure a display rule so that a specific external web application is available in the Application Viewer tab when a system object is selected in the Management View.

  1. The librarian configured the Web Links library.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Client Settings > Web Applications.
  3. Click the Rule Editor tab.
  4. Open the Output expander and do the following:
    a. From the Application drop-down list, select the entry that corresponds to the external web application to display.
    b. (Optional) Modify any application parameters.
  5. Open the Triggers expander and define the conditions to which the display rule applies:
    a. Drag a root node of the Management View from System Browser to the Path field. This creates a new row.
    b. From the Discipline drop-down list, select Fire.
    c. From the Subdiscipline drop-down list, select All.
    d. From the Type drop-down list, select Area.
    e. From the Subtype drop-down list, select All.
    f. Use the equals (=) operators to filter by Path, Discipline/Subdiscipline, and Type/Subtype.
  6. (Optional) To create an override link
    a. Open the Advanced Settings section,
    b. In the Application URL field, type the name of the URL.
    c. In the Username field, type the required username.
    d. In the Password field, type the associated password.
  7. Click Save As .
  8. In the Save Object As dialog box, select the main Web Applications folder or any subfolder under it as the destination where you want to save the new rule.
    a. In the Name field, enter the text DisplayRuleforDMSObjectSelection.
    b. Click OK.
  1. In the General Settings expander, the Application Name field displays the name of the rule. The new display rule object is available in System Browser.
