Configure Modbus Driver and Network for Siport

This procedure is part of the workflow for integrating a SiPort Modbus system.

 

Configure the Modbus Driver for SIPORT

  1. In System Browser, depending on where the driver will run, select one of the following:
    - Project > Management System > Servers > Main Server > Drivers
    - Project > Management System > FEPs > [FEP station] > [drivers folder]
    NOTE:     If the drivers folder does not exist on the FEP station, you can create it using New .
  2. Click New , and select New MODBUS Driver.
  3. In the New Object dialog box, enter a description.
    NOTE: The description must be different from that of any other driver on the Server or FEP stations.
  4. Click OK.
  5. The new driver is added to System Browser, but not started.
  6. Select the driver node and click the Extended Operation tab.
  7. Note down the Driver ID.
  8. Edit the Config text file available under
    [Installation Drive]:\[Installation Folder]\[Project Name]\config
    Add the following lines:
    [Mod_<driverID>]
    maxGap=10
    where <driverID> is the driver ID of the Modbus driver in your project.
    For example, assuming the driver ID is =1, the new config lines must be:
    [Mod_1]
    maxGap=10
  9. In the Extended Operation tab, select the Manager Status property.
  10. The status of a newly created driver is Stopped.
  11. Click Start to activate the driver.
    NOTE: Only for configuration purposes, you can also click Start Conf… to activate the driver without establishing any real connection to the field.

 

Configure the Modbus Network for SIPORT

  1. Select Project > Field Networks.
  2. In the Object Configurator tab, click New , and select New MODBUS Network.
  3. In the New Object dialog box, enter a name and description.
  4. Click OK.
  5. The newly created network is automatically selected in System Browser.
  6. In the Modbus tab, open the Network Settings expander.
  7. In the Monitored by driver drop-down list, select the driver.
  8. Click Save .
