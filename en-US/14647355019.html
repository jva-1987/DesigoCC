CS1140 AlgoRex Troubleshooting

This section contains CS1140 AlgoRex troubleshooting information.

CS11 AlgoRex Import Errors

Message

Situation

Do the following...

The specified file is invalid.

The selected file to import is not a valid configuration file or it is not suitable to the Desigo CC sales channel.

Retry using a new file. Check that the file is generated by a tool belonging to the same Desigo CC sales channel.

There are warnings or errors, look at the log file.

The selected file to import is valid, but some data is not supported.

Open the analysis log (Analysis Log button) to check the pre-import log.

Import cannot be performed. Import would exceed the system load.

Based on the hardware category of the installation, there is a limit to the number of objects a project can have.

The system prevents you from performing imports that would exceed this limit.

The system warns you if an import would approach this limit (90% or more of allowed objects). You can then choose whether to proceed.

If the installation allows for it, you can change the hardware category in the server properties to increase the system objects limit.

 

CS11 AlgoRex Configuration Misalignment

Desigo CC can detect a misalignment with the panel configuration. The information provided in Event List helps you to identify the misalignment.

Event source

Situation

Do the following...

Unidentified

You modified the fire system configuration so that it is no longer aligned with Desigo CC and the NK823x.

Re-import the CS11 system configuration file, and download the configuration to the NK823x.

Problems with CC11 Ack and Reset Commands

Condition

Situation

Do the following...

Cannot Acknowledge and Reset fire events from Algorex fire panels from the Event List.
Ack and reset of the very same objects work fine from System Manager.

If enabled, the BACnet Block Commands can cause this problem.

In the Management View of System Browser, in [Field Networks], select the [BACnet]node and then the BACnet tab. In the Block Commands Behavior extender, deselect all block commands (Ack, Reset, and Silence/Unsilence).

CS11 AlgoRex Migration to FS20 Failure

Message

Situation

Do the following...

Fire objects in the graphics do not migrate. The migration log indicates “Failed; Unable to save graphic”

Cannot write the graphics. There may be a problem in the visibility of graphics in System Browser for the Desigo CC user due to a reduced visibility scope.

If your visibility on the system objects is based on a scope that excludes the graphics, try migrating with another Desigo CC user or with a modified scope.

Fire objects in the graphics do not migrate. The migration log indicates “Failed; Not found”

Misalignment between the CS11 configuration and the FS20 SiB-X file.

Get a FS20 SiB-X file that corresponds with the CS11 configuration and try migrating again.

Problems with Scopes after CS11 AlgoRex Reimport

Condition

Situation

Do the following...

Scopes do not work properly after a reimport of the CS11 AlgoRex configuration with changes to the object structure (CSX numbers changed).

Scopes are not automatically adjusted after these configuration changes.

After a CS11 reimport, check the Scopes where the AlgoRex fire objects are involved. The entries that have changed display in red and must be modified.
