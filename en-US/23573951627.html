Link HDBs on different SQL Servers

In a distributed system, HDBs may be located on different SQL Servers. If you want a project to access data (for example, for reports) from multiple HDBs on different SQL Servers, you must link the SQL Servers.

  1. You have set up the distributed system.
  1. Start SMC.
  2. In the SMC tree, select Database Infrastructure > [SQL Server Name].
  3. Open the Distribution expander
  4. A list of known SQL Servers is displayed. If the desired server is not displayed, you must link it to the SMC.
  5. Select the server you want to link with the server selected in the SMC tree and select the Linked check box.
    NOTE: The Linked check box of the HDB in the Databases expander must also be selected.
  6. In the SMC tree, select the SQL server you linked in step 4.
  7. Open the Distribution expander.
  8. Select the server instance with the other HDB and select the Linked check box.
    NOTE: Both servers must be linked to each other.
  9. Click Save.
