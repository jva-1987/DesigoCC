MM8000 Migration Troubleshooting

This section contains MM8000 migration troubleshooting information.

Handling MM8000 Graphic Import Errors

Message

Situation

Do the following...

The Import Summary indicates import warning or failures:

Graphics imported with warnings: …

Graphics failed to import: …

Geographical trees imported with warnings: …

Geographical trees failed to import:

Errors occurred in importing some MM8000 maps/ geographical trees.
Warnings indicate that errors could be recovered. Instead, import failed messages mean that one or more maps could not be imported at all.

See the detailed log files:

<drive>:\Users\<username>\AppData\Local\Temp\MM8000Import_mm-dd-yyyy_nn.txt
and

<drive>:Users\<username>\AppData\Local\Temp\MM8000Import_Results yyymmdd hhmmss.txt)

Fix the possible graphic errors using the management platform tools.

The version of the Composer project you are importing is not supported.

You tried to import an MM8000 project older than 4.50.

You cannot import projects older than 4.50. Update the project using a newer MM8000 version and try again.

A dialog box displays and prompts you to:

Select the correct association of the MM8000 Point.

The import procedure cannot unambiguously identify the management platform point to associate to an MM8000 point represented on a map.

The dialog box presents the MM8000 subsystem and point names and a list of management station points that included the extended location paths.

Select the management station point you want to associate to the MM8000 points and click OK, or click Cancel to skip importing the point.
