Configuring the Scope for the Alarm Suppression Visualization

To notify the operators that the alarm suppression feature is active in the system, the Event Manager object will generate an event when at least one system object is affected by alarm suppression. To allow the operators to see the event relevant to the alarm suppression activation, users groups must be properly configured with an alarm suppression scope.

  1. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Scopes.
  3. The Scopes tab displays.
  4. From System Browser, drag the Event Manager object to the Scopes field of the Scope Rules section.
    NOTE: The Event Manager object is located at the following path:
    Project > Management System > Servers > Main Server.
  5. The Scopes field is automatically set based on the linked object.
  6. Click Save .
  7. In the Save Object As dialog box, enter a name and description (for example, Scope_AlarmSuppression) and click OK.
  8. Select Project > System Settings > Security.
  9. The Security tab displays. At least one user group is available in the Groups list.
  10. In the Groups list, select the desired user group.
  11. In the Scope Rights expander, do the following:
    a. From System Browser, drag the alarm suppression scope to the Scope field.
    b. Specify the following settings:
    - Object Type
    - Property Groups
    - Command Groups
  12. Click Save .
