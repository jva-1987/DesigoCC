SiPass Access Control Events

Access control events are in general for monitoring purpose. In Desigo CC Event List, the symbol will show if the event coming from the access control system requires operator acknowledgment. No reset command is required. For more background information, see the event treatment section.

The table below sets out the event handling method for different types of SiPass events:

Type of Access Control Event

Event Handling

Unauthorized or invalid access control events coming from SiPass

Desigo CC operator acknowledgment is required

Authorized or valid access control events coming from SiPass

No event shows on Desigo CC Event List

Operator manual control actions from Desigo CC, or from the SiPass server, such as lock, unlock, block, unblock.

No acknowledgment required. The manual control event displays on Desigo CC and will be removed when the manual control action is reverted.

Hardware fault or off-normal conditions from the SiPass system

Operator acknowledgment is required. The event will be removed when the fault or off-normal condition is restored.

SiPass Flag status change

No acknowledgment required. If the flag is configured for Audit Trail Reporting, an event shows in the Desigo CC Event List when the flag status changes to True and is removed when the status reverts to False. For details, see SiPass Flags.
