Manually Creating a New Library

You can manually add a new library to a Desigo CC project. For background information, see Library Configurator Workspaces. This procedure covers how to create a generic new library from scratch. Another way to create a new library is by cloning the structure of an existing library, and then customizing it (see Creating a Customized Library).

 

  1. You are trained and authorized to work with system libraries at the target customization level. For example, L4-Project.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L4-Project or L3-Region or L2-Country.
  3. In the Library Configurator tab, click New and select New Library Object.
  4. In the General Settings expander, specify the following fields.
    Asterisks (*) indicate the mandatory ones. The Discipline, System and the main Version number become read-only after you create the library:
    • Discipline: If you want to create a library directly in a discipline-specific folder, enter the name of the folder in this field (for example, Global). If you want to create a library object in a subfolder of a discipline-specific folder, enter the name of the folder as follows:
      [Discipline-specific]_[subfolder] (for example, Global_mysubfolder).
    • System
    • (Optional) Version
    • (Optional) Level tag: available only for L4-Project library objects
    • (Optional) Date
    • (Optional) Languages
    • (Optional) Dependencies: From the drop-down list, select any libraries on which this new library depends, and click Add to add them to the list. To remove a library from the list, select it and click Remove.
  5. Click Save .
  6. In the New Object dialog box, enter a name and description, and click OK.
  7. The new library is added to System Browser under the discipline folder or subfolder that you specified.
  8. You can now proceed to add any blocks that you want the new library to have. See Adding Blocks to a Library.
