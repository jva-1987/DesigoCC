Resolve Inconsistencies during Assignment

The following are procedures for resolving inconsistencies to assignments.

Trend log Object is False

  1. In the Data import expander, select the Trend channels tab.
  2. Enter the following migration information:
    a. Select one or more trend channels and select the check box.
    b. In the Data from field, enter the start date.
    c. In the Data To field, enter the end data.
    d. In the In archive group drop-down list, select the archive group. By default the data in the archive group Value Log Archive Schedule is imported.
  3. Perform an import.
  4. The data is saved to an offline trend log object after import.
  5. Double-click the data point in the CC Trended Property column.
  6. The data point is selected in System Browser.
  7. Select the Related items tab and then Trend.
  8. Click the corresponding trend label.
  1. The trend data is displayed in the secondary pane.

 

Check is Not Possible

Cause: In Desigo Insight, two or more data points were assigned to the same trend log object while trending.

  1. In the Data import expander, select the Trend channels tab.
  2. Enter the following migration information:
    a. Select one or more trend channels and select the check box.
    b. In the Data from field, enter the start date.
    c. In the Data To field, enter the end date.
    d. In the In archive group drop-down list, select the archive group. By default, the data in the archive group Value Log Archive Schedule is imported.
  3. Perform an import.
  4. The data is saved to the database after import.

Continued Trending of Data

  1. Double-click the data point in the CC Trended Property column.
  2. The data point is selected in System Browser.
  3. Select the Object Configurator tab.
  4. In the Properties expander, select the Property column.
  5. Select the Property and select the VL check box.
  6. Click Save .
  1. All data continues to be recorded.

 

No Assignments Available

Cause:

There is no corresponding data point in Desigo CC. The data point may have been renamed in advance in Desigo CC.

Error correction:

The altered Desigo CC data point must be manually supplemented if Desigo Insight data is required.

  1. In System Browser, select logical view.
  2. Select the Manual Navigation check box.
  3. Select the appropriate data point.
  4. Drag the data point to the CC Trended Property column for the corresponding entry.
    Note: Assignments are lost when changing applications and must be reassigned.
  5. Select the check box.
  6. Perform an import.
  1. The data is saved to the database after performing the import. The data is no longer automatically recorded. Select the property values log for the data if data still must be recorded.

Continued Trending of Data

  1. Select the Manual Navigation check box.
  2. Double-click the data point in the CC Trended Property column.
  3. The data point is selected in System Browser.
  4. Select the Object Configurator tab.
  5. In the Properties expander, select the Property column.
  6. Select the Property and select the VL check box.
  7. Click Save .
  1. All data continues to be recorded.
