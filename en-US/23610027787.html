Repair the Project Database of the Stopped Project

  1. You selected a project in the SMC tree with a corrupted project database indicated by the status stopped - last repair failed.
  1. In the Project Settings toolbar, click Repair on next start .
  2. On successful repair, the project status changes to stopped - repair on next start and the Repair on next start icon disappears.
  3. Click Start Project . Either of the following may happen:
  4. The project database is successfully repaired and started. However, it may take a little longer for the project to start, as it internally attempts to repair the corrupted database.
  5. The project fails to start. This might be because the database is corrupted beyond repair. One way to further attempt a fix is to remove the events from the project due to the possibility of the alarm part of the project database being corrupt. To do this, click Remove Events .
  6. Start the project by clicking Start Project . The system will make another attempt to repair the database.
