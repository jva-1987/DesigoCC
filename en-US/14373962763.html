Customizing OPC Discovery Rules

You can configure the rules of the OPC discovery library that will be used when importing OPC Items. Instead of configuring a new OPC discovery rules library from scratch, you may want to create a customized version of an existing one by cloning it at your allowed customization level. For background information see, Third-party OPC DA Discovery Rules Workspace.

  1. You are trained and authorized to work with system libraries. Also you are authorized to handle libraries for the specific customization level (L4-Project, L3-Country, or L2-Region) where you want to create the OPC discovery rules library.
  2. System Manager is in Engineering mode.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L1-Headquarter > Global > OPC > [OPC discovery rules folder] > [OPC discovery rules library].
  3. In the Library Configurator tab, click Customize .
  4. Click OK to confirm.
  5. The selected library is cloned under the allowed customization level (for example, L4-Project > Global > OPC >[OPC discovery rules folder] > [OPC discovery rules library]).
  6. (Optional) You can now continue with modifying the hierarchy settings, creating new object mapping rules, or removing and moving up or down any rule that belongs to the current level. See Configuring the Settings for an OPC Discovery Rules Library.
    NOTE: You cannot change the Library Name and Match String character in the Library Settings expander. Also, you cannot modify the mapping rules defined at upper customization levels.
