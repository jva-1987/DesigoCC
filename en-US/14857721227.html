Haystack Commands, Events and Library Objects

The following describes commands, events, and library objects of the Haystack integration.

Haystack Commands

You can command Haystack objects with writable markers. Do the following steps:

  1. Select the object in System Browser.
  2. In the Extended Operation tab, select Write Level and set the command Priority (from 1=highest to 17).
  3. Select Write Value and set the new value that can be:
    • For boolean points, 0 or 1
    • For numeric points, a number in the expected range
    • For strings, a number corresponding to the string value in the State List
      NOTE: Haystack string points have a numeric State Value property in Desigo CC that corresponds with one of the strings of the State List.

For details about the SORIS adapter properties and commands, in SORIS, see Properties and Commands of SORIS thingAdapter Objects.

Haystack Events

The following is the list of events available in Desigo CC for the Haystack integration:

  • Adapter connection fault (communication between the adapter and the Haystack subsystem)
  • Watcher information (diagnostic status on the Haystack subsystem)

Specific object events can be configured as Desigo CC management station alarms.

Haystack Library

The BA_Haystack_150_HQ_1.gms library provides object models, functions and graphic symbols to manage Haystack entities in Desigo CC. Refer to the BA Device section in system libraries.
