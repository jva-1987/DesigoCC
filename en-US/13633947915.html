Restricting Access to a System Browser View

By default, System Browser views are available to all the users configured for a project. You can create a scope rule that restricts access to a view to a specific user group.

  1. System Manager is in Engineering mode.
  2. You previously configured users and at least one user group.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Scopes.
  3. The Scopes tab displays.
  4. In System Browser, select the Manual navigation check box.
  5. Select the view you want to restrict from the drop-down list.
  6. Select the root node of the view and drag-and-drop it to the Scope Rules area.
  7. The hierarchical path of the linked object is available as a scope rule in an Include row.
  8. Click Save.
  9. In the Save Object As dialog box, enter a name and description for the scope rule, and click OK.
  10. Select Security.
  11. The Security tab displays.
  12. In the Groups area, select the user group that will have access to the view defined in the scope rule.
  13. The configuration expanders refresh to display the settings for that user group.
  14. From System Browser, drag-and-drop the previously created scope rule to the Scope Rights area, and define the rights.
  15. Click Save .
  1. The view configured in the scope rule is only available for the users belonging to the user group defined for the scope rights.
