Preparing the CSV Configuration Files for the Third-party OPC Server

To be able to integrate third-party OPC devices (OPC servers) into Desigo CC, you must first manually create the configuration files in CSV (comma separated values) format, using Microsoft Excel or a text editor.

Use a semicolon (;) as separator of the values contained in a CSV file.

Create an OPC Object Model Configuration file (CSV)

If the object models provided in the default OPC library are not sufficient you must create this file that defines the custom object models that you require (see CSV File for Object Models). You will use this file when you import the custom OPC object models into the OPC library (see Importing Custom OPC Object Models into the OPC Library).

 

Create an OPC Devices Configuration file (CSV)

The OPC devices configuration file defines the OPC server-groups-items structure for the field devices/subsystems that you want to integrate (see CSV File for Third-party OPC Devices). You will use this file when you import the OPC items into the OPC network (see Importing Third-party OPC Server Data).

 

Convert the Format of an OPC Configuration File

To import an older version of an OPC CSV file you have to first convert it to the latest format using the OPC Conversion Tool.

  1. On the computer, select [Installation Drive]:\[Installation Folder]\GMSMainProject\bin.
  2. Double-click Siemens.Gms.Opc.Csv.Converter.exe.
  3. In the command prompt window, drag the CSV file to be converted.
  4. Press ENTER.
  1. A new OPC configuration file is created with the same name and suffix _converted.
