Haystack Troubleshooting

This section covers several areas of troubleshooting.

Troubleshooting SORIS

In SORIS, see SORIS Troubleshooting.

Handling Connection Error Conditions for Haystack

Issue

Situation

Do the following...

The adapter cannot reach the Haystack website.

The connection is lost because your internet connection or the Haystack website is down.

  1. Check your internet connection.
  2. Check the Haystack website connection:
    a. In System Browser, select Project > Field Networks > [SORIS network] > [Haystack adapter] > Configuration > Adapter Settings.
    b. In the Operation tab, check the Service Status property. If Disconnected, wait until the connection is re-established and the Service Status property becomes again Connected.

In the Operation tab, the Service Status property for the adapter indicates Disconnected.

The adapter is not running or the connection between Desigo CC and the configured URL is not working.

  1. Check that the Haystack adapter service is running.
  2. Check the IP address and the port number.

After the adapter configuration and discovery are carried out, the objects are not created in System Browser.

The configuration files are invalid.

Check the configuration files ( see Installing and Configuring the Haystack Adapter):

  1. Check that the config files are located in the same directory as the Haystack adapter executable file.
  2. Check the content of config files.
  3. Stop the adapter service, then start the adapter in console mode and check for error messages.
