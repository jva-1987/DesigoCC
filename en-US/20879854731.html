Configuring Client Profiles

The client profile is a configuration that can be applied to an individual user or station. It customizes certain aspects of the Desigo CC user interface, and how events are reported. In particular:

  • The 'standard' Client Profile applies to Desigo CC installed client and FEP stations. For more details, see Standard Client Profiles.
  • The Flex Client Profile applies to Flex Client web stations. For more details, see Flex Client Profiles.

If a client profile is has been set for a station, it overrides any client profile defined for the user.

 

Define appropriate profiles: either for all stations, or for all users

By default Desigo CC and Flex Client start with no profile associated to stations, and with DEFAULT profiles associated to the users. These are generic profiles not intended for operational use. Therefore, make sure to do one of the following:
- Define appropriate profiles for all the stations. These station profiles will override the user's individual profile settings.
- If you omit the station profiles, make sure to define appropriate profiles for all users.

If you are unsure which profile to select, see Determine which Client Profiles are Compatible.

Prerequisites

  • System Manager is in Engineering mode
  • System Browser is in Management View.

Set the Client Profiles for a User

If there is no client profile defined for a Desigo CC station, then the client profiles configured here for the user will apply.

  1. In System Browser, select Project > System Settings > Users.
  2. Select the Users tab.
  3. From the Users list, select the user whose client profiles you want to configure.
  4. In the User Settings expander:
    • From the Client profile drop-down list, select the profile to apply for the installed Desigo CC application.
    • From the Flex client profile drop-down list, select the profile to apply for the Desigo CC web application.
  5. Click Save .
  1. The configured profiles will apply when the user logs onto a station with undefined client profile.

 

NOTE: In Flex Client, if the profile of an existing user is changed, to force the correct columns display in Event List do the following:

  1. Log on to Flex Client with the user’s credentials.
  2. In Event List, select  Restore to default.
  1. Event List displays with the columns depending on the user’s new profile settings.

 

Set the Client Profile for an Installed Station

If you define a client profile for a Desigo CC station, it will take precedence over the profile defined for the user.

  1. In System Browser, select the station at one of the following paths:
    • server station: Project > Management System > Servers > Main Server
    • installed client: Project > Management System > Clients > [client]
    • installed FEP client: Project > Management System > FEPs > [fep]
      NOTE: If the object representing the [client] or [fep] is not present, see Creating Stations in System Manager.
  2. Select the System Management tab.
  3. From the Client profile drop-down list, select the profile that you want to apply.
  4. Click Save .
  1. The client profile is set on the station. It will apply to all users who log onto that station.

 

Set the Client Profile for a Flex Web Station

The profile set for the anonymous Flex Client will apply collectively to all the Flex Client stations that connect to Desigo CC as anonymous web clients.
On the other hand, you can apply a distinct profile to each identified Flex Client station. In either case, the defined station profile takes precedence over any profile defined for the user.

  1. In System Browser, select the station at one of the following paths:
    • anonymous Flex Client: Project > Management System > Clients > Default Web Client
    • identified Flex Client: Project > Management System > Clients > [identified Flex Client]
      NOTE: If the object representing the [identified Flex Client] is not present, see Creating Stations in System Manager.
  2. Select the System Management tab.
  3. From the Flex client profile drop-down list, select the profile that you want to apply.
  4. Click Save .
  1. The client profile is set on the station. It will apply to all users who log onto that station.

 

Work with LDL Profiles

 

Working with LDL profiles extensions
DEFAULT.ldl is the base client profile starting from which LDL extensions can be created to contain any addition, replacement, and deletion entity with respect to the base LDL profile.
Desigo CC loads the base profile (DEFAULT.ldl) or one of its extensions. Desigo CC uses the name of base profile string (that is, DEFAULT.ldl) to search for a folder in the profiles folder belonging to the current project. If it finds a folder with this name (that is, DEFAULT), it loads all the LDL files present in this folder that represent any extensions of the DEFAULT client profile. Then Desigo CC merges the base client profile with the extensions.

To avoid inconsistencies:
- Do not rename the DEFAULT.ldl or the DEFAULT folder.
- Make sure that all the LDL extensions are included in the DEFAULT folder.

 

Working with customized LDL profiles
To work with the customized LDL profiles in the current version, you must save those profiles using a unique name other than the profile name of the installed extensions and then you must add them in a allow-list file (CustomizedProfiles.ini file at the location ...\GMSMainProject\Profiles) before restoring or upgrading the project. Customized profiles not included in the CustomizedProfiles.ini file can be deleted during SMC operations.
For projects earlier than V3.0, you must manually create the file CustomizedProfiles.ini by adding the customized profiles.
