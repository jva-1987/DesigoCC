Controlling Recipients

This section explains how to add, update, remove, import and export a recipient user, and recipient group.

For background information related to recipients and recipients workspace, see Recipients.

 

Add a Recipient User

  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. Click Recipients.
  4. Click Edit .
  5. Click Add User .
  6. Configure the parameters in the General, Recipient User Devices, and Group Membership expanders.
  1. The recipient user is added and configured.

 

Add a Recipient Group

  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. Click Recipients.
  4. Click Edit .
  5. Click Add Group .
  6. In the General section, specify the name of the recipient group.
  7. Open the Members expander and click Add to add recipients to the group. You can add maximum 1000 recipients in one recipient group.
  8. Open the Groups Membership expander and click Add.
  9. The Groups Membership dialog box displays.
  10. Select the recipients to be associated with the group, and click to move the selected recipients to the text box on the right.
  11. Click OK.
  12. The Members list is updated with the selected recipients.
  13. Open the Groups Membership expander and click Add.
  14. In the Groups Membership dialog box, associate a recipient user with the configured recipient groups.
  15. Click OK.
  16. Click Save .
  1. The group is created and users are associated with the group.

 

Browse Recipients

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. All the available individual users and recipient groups display.
  4. Select the Recipients tab and do the following:
    a. Click Filter , and select the user, or groups check boxes, depending on your specific search.
    b. Enter a keyword in the Search field to search a specific recipient.
    NOTE:     Search terms must not contain the word search (even as part of a word) or the _ (underscore) character.
  5. Select the recipient to be viewed.
  1. The respective recipients’ details display. Further recipients can be added, updated and removed.

 

Update Recipients

  1. The recipient profile to be updated is present in Applications > Notification > Recipients.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. A list of all the available recipients displays.
  4. Select the recipient profile to be updated. To narrow the search, do the following:
    a. Click Filter and select the User or Groups check boxes, depending on the search.
    b. Enter a keyword in the Search field to search a specific recipient.
  5. The respective recipient details display. However, the Add and Remove buttons remain disabled.
  6. Click Edit to enable the Add and Remove buttons.
  7. Select the General expander and edit the required fields (see Add a recipient user section).
  8. Select the Recipient User Devices expander.
  9. Select a device type and click Edit to modify the information.
  10. Open the Groups Membership expander.
  11. Click Add to introduce a new group or click Remove to remove the existing group membership.
  12. Click Save .
  1. The recipient details are updated.

 

Remove Recipients

  1. A recipient profile to be removed is available under Applications > Notification > Recipients.
  2. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. A list of all the available recipients displays.
  4. Select the recipient profile to be removed.
  5. (Optional) To narrow your search, do the following:
    a. Click Filter and select the User or Groups check boxes, depending on the search.
    b. Enter a keyword in the Search field to search for a specific recipient.
  6. The respective recipient details display. However, the Add and Remove buttons remain disabled.
  7. Click Edit to enable Add and Remove buttons.
  8. Click Remove .
  9. A confirmation dialog box displays.
    NOTE 1: The deleted recipient user cannot be used for new messages and is removed from all existing message templates. However, the users are listed in the active and expired messages for an accurate historical view.
  10. Click Yes.
  1. The recipient is deleted.

 

Import Recipients

You can import recipients using either of the following:

Import Recipients from XML

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients tab displays.
  4. Select the Import tab.
  5. Click Recipient import edit to enable import options.
  6. Open the Import from XML expander.
  7. The Open dialog box displays.
  8. Click Browse.
  9. Select the location where the XML file is stored.
  10. Select the XML file.
  11. Click Open.
  12. The path of the XML file is displayed in the Import file field.
  13. Click Import.
  1. The selected XML file is imported.

Import Recipients from XML using Commands

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients tab displays.
  4. Select the Operation tab.
  5. Click Import XML link to enable the import options.
  6. The File Path field and Send button display.
  7. In File Path field, enter the local XML file path of the server.
  8. Click Send.
  1. The selected XML file is imported.

Import Recipients from CSV

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients tab displays.
  4. Select the Import tab.
  5. Click Recipient import edit to enable import options.
  6. Open the Import from CSV expander.
  7. The Open dialog box displays.
  8. Click Browse.
  9. Select the location where the CSV file is stored.
  10. Select the CSV file.
  11. Click Open.
  12. The path of the CSV file is displayed in the Import file field.
  13. Select the Delete absent recipients check box to delete the recipients in the Notification application that are not present in the CSV file.
    NOTE 1: Recipients created manually through the recipient UI will never be deleted during CSV import.
    NOTE 2: The mapping name is used as one of the criteria to delete recipient users. Hence, when the user deletes the mapping, a warning message displays.
  14. Click Import.
  1. The selected CSV file is imported.

NOTE:
If imported recipient users need to be updateable through subsequent imports, the unique ID property under Notification property should be mapped to a CSV column that contains values that uniquely identify recipient users, like an employee ID.

 

Import Recipients from CSV using Command

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients tab displays.
  4. Select the Operation tab.
  5. Click Import CSV link to enable the import options.
  6. The Delete Users, Mapping, File Path fields and Send button display.
  7. Select the appropriate value (Yes or No) from the Delete Users check box.
  8. Enter the mapping to be used for the CSV import in the Mapping field.
  9. In File Path field, enter the local CSV file path of the server.
  10. Click Send.
  1. The CSV file specified in the file path is imported.

Import Recipients from Active Directory

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients Editor tab displays.
  4. Select Import tab.
  5. Click Recipient import edit to enable import options.
  6. Open the Import from Active Directory expander.
  7. Enter the Active Directory server's address in the Server field.
  8. Click Connect.
  9. The list of the departments available in the corresponding Active Directory displays in the Available departments section.
  10. Select the departments to be imported.
    NOTE: Search for the department by typing in the department’s name in the Search field.
  11. For importing departments, click to move the department to the Departments to import section.
  12. To move departments from the Departments to import section to the Available departments section, and click to move the department to the Available departments section.
  13. Select the Delete absent recipients check box to delete the recipients in the Notification application that are not present in the Active Directory.
    NOTE 1: Recipients created manually through the Recipient UI will never be deleted during Active Directory import.
    NOTE 2: The mapping name is used as one of the criteria to delete recipient users hence when user deletes the mapping, a warning message displays.
  14. Click Import.
  1. The selected departments are imported.

NOTE:
If entries imported from the Active Directory match existing entries, details of the corresponding recipient users are updated according to the details specified in the corresponding department of the Active Directory.

Import Recipients from Active Directory using Command

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients Editor tab displays.
  4. Select the Operation tab.
  5. Click Import AD link to enable the import options.
  6. The AD Server, Delete Users, Departments, Mapping fields and Send button display.
  7. Enter the Active Directory server address in the AD Server field.
  8. Select the appropriate value (Yes or No) from the Delete Users list to delete recipients from Notification that are not present in the Active Directory.
  9. Enter the department name to import. Multiple department names can be entered by using a comma separator.
  10. Enter the mapping to be used for the Active Directory import in the Mapping field.
  11. Click Send.
  1. The selected departments are imported.

 

Export Recipients

You can export recipients using either of the following methods:

Export Recipients to XML

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients Editor tab displays.
  4. Select the Export tab.
  5. Click Recipient export edit to enable export options.
  6. Expand the Export to XML section and select the recipients to be exported and click to move the selection to the Selected for export text box.
  7. Click Export.
  8. The Save dialog box displays.
  9. Select the location where the XML file must be saved.
  10. Enter a File Name and click Save.
  1. The recipients are exported in XML format.

Export Recipients to XML using Command

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients tab displays.
  4. Select the Extended Operation tab.
  5. Click Export XML link to enable the export options.
  6. The File Path field and Send button display.
  7. In File Path field, enter the local file path of the server where the XML file must be saved.
  8. Click Send.
  1. The recipients are exported to the XML file specified in the file path.

Export Recipients to CSV

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients Editor tab displays.
  4. Select the Export tab.
  5. Click Recipient export edit to enable export options.
  6. In the Export to CSV expander, do the following:
    a. Select the recipients to be exported.
    b. Click to move the selection to the Selected for export text box.
  7. Select a mapping configuration if present or create a mapping configuration. For more information, see Configuring Mappings for CSV Export in Configure Mappings from Controlling Recipients section.
  8. Click Export.
  9. The Save dialog box displays.
  10. Select the location where the CSV file must be saved.
  11. Enter a file name and click Save.
  1. The recipients are exported in CSV format.

Export Recipients to CSV using Command

  1. System Manager is in Operating mode.
  1. In System Browser, select Application View.
  2. Select Applications > Notification > Recipients.
  3. The Recipients Editor tab displays.
  4. Select the Extended Operation tab.
  5. Click Export CSV link to enable the export options.
  6. The Mapping, File Path field and Send button display.
  7. Enter the mapping to be used for the CSV export in the Mapping field.
  8. In File Path field, enter the local file path of the server where the CSV file must be saved.
  9. Click Send.
  1. The recipients are exported in CSV format.

 

Configure Mappings

You can configure mappings for importing or exporting CSV files or for an active directory import.

Create a New Import Mapping

  1. Before configuring mapping for importing CSV files, you first need to determine the properties of the CSV files.
  1. Click New.
  2. The Create mapping as dialog box displays.
  3. Enter a name for the mapping in the Create mapping as dialog box.
  4. Click OK.
  5. The Mapping field is populated with the name specified for the mapping.
  6. Select the Header row check box if the user wants to specify a title for the CSV file.
  7. Select a separator from the Delimiter drop-down list.
  8. In the Columns section, map a CSV column with a notification property.
  9. For mapping, select a CSV column option from the CSV Column drop-down list and select a notification property for the corresponding CSV column option from the Notification Property drop-down list. For example, Column 1(A) of CSV Column can be mapped to First Name of the Notification Property. Correlate the alphabet A from the mapping column with the Microsoft Excel column header A, if Microsoft Excel is used to view the CSV file.
  10. Assign an individual recipient to recipient group by selecting the appropriate CSV column from the CSV Column drop-down list and mapping it to the recipient group from the Notification Property drop-down list.

  11. NOTE 1:
    The recipient group to be used during import of CSV file must be first created in the Notification application.
    NOTE 2:
    The individual recipient will not be assigned to the recipient group if the corresponding value in the CSV is either of the given values—
    "", " ", "0", "NO", or "FALSE"
    For values other than those mentioned above, the individual recipient will be assigned to the recipient group.
    NOTE 3:
    To edit an existing mapping, select the mapping, perform the required edits and click Save. To save the mapping with a new name, click Save As and provide a new name for the mapping.

  12. Select a quotation type from the Quotations drop-down list.
  13. Click Save.
  14. The mapping is configured and can be saved for future use.

Configure Mappings for an Active Directory Import

  1. Click New.
  2. In the Create mapping as dialog box that displays, enter a name for the mapping.
  3. Click OK.
  4. The Mapping field is populated with the name specified for the mapping.
  5. In the Columns section, map an Active Directory Property with a Notification Property.
  6. Select an Active Directory Property from the drop-down list or enter manually in the Active Directory Property field.
    For example, First name of the Active Directory Property can be mapped to First Name of Notification Property.
    NOTE:     Each Active Directory property has a real name and a display name, for example, givenName is the real name of an Active Directory property with display name as First name. The properties listed in the Active Directory Property drop-down list are the display names of the real Active Directory properties. You can enter the real name of the Active Directory Property by checking for its corresponding display name. For example, a tool called Remote Server Administration Tools for Windows 7 shows the real Active Directory property name and its display name to be used.
  7. Click Save.
    NOTE: An error message displays if there are invalid entries in the Active Directory properties mapping. In such scenarios, click OK. Rectify the error and click Save again.
  8. The mapping is configured.
  9. To edit an existing mapping, do the following.
    a. Select the mapping from the Mapping drop-down list.
    b. Edit the required fields.
    c. Click Save.
  1. The mapping is updated.

Configure Mappings for CSV Export

  1. Click New.
  2. The Create mapping as dialog box displays.
  3. Enter a name for the mapping and click OK.
  4. The name is displayed under Mapping.
  5. Select the Header row check box if the exported CSV file must contain a header row.
  6. Select a separator from the Delimiter drop-down list.
  7. In the Columns section, to map a Notification Property with a CSV Column. Do the following:
    a. From the Notification Property drop-down list, select a notification property.
    b. In the CSV Column drop-down list, enter a CSV column option for the corresponding notification property.
  8. From the Quotations drop-down list, select a quotation type.
  9. Click Save.
  1. The mapping configuration is saved.
