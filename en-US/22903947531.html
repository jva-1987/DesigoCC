Configure Server-specific DCOM Settings

Once the system-wide DCOM settings are properly configured, it is necessary to configure the server-specific DCOM settings. To modify the configuration, do the following:

  1. In the Windows search box on the taskbar, enter DCOMCNFG.
  2. Press ENTER.
  3. The DCOM configuration process is initiated.
  4. In the Component Services window, under Console Root, expand Component Services, and then expand the Computers folder.
  5. My Computer is in the Computers folder.
  6. Expand My Computer and select DCOM Config.
  7. In the list of objects in the right window pane, right-click [company name].OPC.Server.DA and select Properties.
  8. The OPC DA Server Properties dialog box displays. In the OPC-server-specific settings, you must change only the Identity tab settings. The rest of the tabs can refer to the default configuration previously set (see Configure Default System-wide DCOM Settings).
  9. In the OPC DA Server Properties dialog box, select the Identity tab.
  10. Select the This user option.
  11. In the User field, enter the User Account name created for OPC (for example, OPCUser).
  12. Enter the password and confirm it.
  13. Click Apply.
  14. Click OK.
