Creating Scripts

For general information on how to create a script, see Step-by-step > Configuring a Project > Configuration Workflows & Procedures > Configuring Scripts.

Copy an Example from the Library

  1. The Scripts extension module is installed.
  2. Two virtual data points are created for calculated values (for example: Average of two OPC data points). Refer to the Virtual Objects description for additional information.
  1. In System Browser, select Management View.
  2. Select Project > System Settings > Libraries > L1-Headquarters > BA > Software > Desigo Gateway Functionality Scripts > Scripts.
  3. Select the Script Editor tab.
  4. The example scripts SXOpenMainScript, SXOpenFunction, SXOpenParser are displayed.
  5. Select the SXOpenMainScript.
  6. Open the Viewer expander.
  7. Select the Script Editor tab.
  8. Press CTRL+A to highlight all the content and CTRL+C to copy it.

Creating and Saving a Script

  1. A new folder is created to save the Gateway Script.
  1. In System Browser, select Application View.
  2. Select Applications > Logics > Scripts.
  3. Select the Script Editor tab.
  4. Open the Editor expander.
  5. Delete the default entries.
  6. Press CTRL+V to paste content.
  7. Click Save As .
  8. In the Save object as dialog box, enter the name and description.
  9. Click OK.

When Does the Script Path Need to be Changed

The script path must be changed if the Scripts SXOpenFunction and SXOpenParser are modified for a project. The variable ConsoleOutput=true displays the results in the Console expander.

Apply SXAdd Function with COV

The examples in the library were created for the communications from OPC to BACnet. The value (3) in Present_Value must be changed for communications from BACnet to OPC.

  1. The OPC and BACnet data points are imported.
  2. The script is opened in System Manager in the Editor expander.
  1. In System Browser, select logical view.
  2. In the Editor expander, select the predefined function SXAvg and copy the template to the script.

Data Points Configuration

  1. Enter a unique subscription number (1) as a description.
  2. Drag the OPC data point from System Manager to the first subscription (2).
  3. (Optional) Another property can be defined instead of the default property Value (3).
  4. Enter the function using the corresponding description number (4). This number must be unique throughout the script. The Script editor does not check the numbers for uniqueness.
  5. Do not change the entry false (5).
  6. Drag the BACnet data point from System Manager to the first subscription (6).
    NOTE: A virtual data point can be assigned instead of a BACnet data point.
  7. Enter the calculation function (7) SXAdd.
  8. Enter an offset (8) to the data point value.

Editing Additional Data Points

  1. Create another SXAdd subscription or select a new script function.
  2. Repeat the steps as described in Data Points Configuration.
  3. Delete unneeded functions from the script.

Applying SXAvg Function with COV

The description is based on three OPC data points.

  1. The preconditions are described in SXAdd.
  1. In the Editor expander, select the predefined function SXAvg and copy the template to the script.
  2. Enter a unique subscription number (1) as a description.
  3. Enter a unique function number (2).
  4. Drag the OPC data point from System Manager to the first subscription (3).
  5. (Optional) Another property can be defined instead of the default property Value (4).
  6. Enter the function using the corresponding description number (5). This number must be unique throughout the script. The Script editor does not check the numbers for uniqueness.
  7. Do not change the entry false (6).
  8. In variable 1, enter the parameter (7) for the subscription data point (3).
  9. Drag the OPC data point from System Manager to the second variable (8).
  10. Drag the OPC data point from System Manager to the third variable (9).
  11. Drag the BACnet data point from System Manager to the first subscription (10).
    NOTE: A virtual data point can be assigned instead of a BACnet data point.
  12. Enter the calculation function (11) SXAvg.
  13. Enter the number (12) of data point values for calculation.
  14. The number of assigned data point variables (13) must match the numbers of variables (12). The remaining variable must be set to 0.

NOTE:
The average is only calculated if the value of the first subscription changes. Use the polling function to perform continuous averaging.

Applying the SXBetween Function with Polling

The examples in the library were created for communications from OPC to BACnet. The value (4) in Present_Value must be changed for communications from BACnet to OPC.

  1. The preconditions are described in SXAdd.
  1. In the Editor expander, select the predefined function SXBetween and copy the template to the script.
  2. Enter a unique function name (1) as a description. The function name must be unique throughout the script.
  3. Enter the polling rate (2) in milliseconds. All data points assigned to this function are polled at the same polling rate.
  4. Drag the OPC data point from System Manager to the first read line (3).
  5. (Optional) Another property can be defined instead of the default property Value (4).
  6. Drag the BACnet data point from System Manager to the first read line (5).
  7. Enter the calculation function (6) SXBetween.
  8. Enter the minimum and maximum limit value (7) of the data point value for calculation.
  9. Create additional data points (lines 2 to 4) in this function or create a new function with a different polling rate.
