URVE Room Booking Troubleshooting

This section covers several areas of troubleshooting.

Troubleshooting SORIS

In SORIS, see SORIS Troubleshooting.

Handling Connection Error Conditions for URVE Room Booking

Issue

Situation

Do the following...

The URVE adapter cannot reach the URVE Room Booking website.

The connection is lost because your internet connection or the URVE Room Booking website is down.

  1. Check your internet connection.
  2. Check the URVE Room Booking website connection:
    a. In System Browser, select Project > Field Networks > [SORIS network] > [URVE adapter] > Configuration > Adapter Settings.
    b. In the Operation tab, check the Service Status property. If Disconnected, wait until the connection is re-established and the Service Status property becomes again Connected.

In the Operation tab, the Service Status property for the adapter indicates Disconnected.

The adapter is not running or the connection between Desigo CC and the configured URL is not working.

  1. Check if the adapter is running.
  2. Check the IP address and the port number.

After the adapter configuration and discovery are carried out, the objects are not created in System Browser.

The configuration file is invalid.

Check the configuration file:

  1. Select Adapter Settings in System Browser. In the Operation tab, check that the full configuration file path is indicated next to the Configuration File property.
  2. Check that the adapter.config file is located in the same directory as the URVE adapter executable file.

The comfort control command before a room occupancy event is not executed

The comfort control command is not configured or the execution script is not running.

  1. In System Browser, select the room and check the configuration in the Operation tab. See Configuring Commands for URVE Room Booking.
  2. In addition, make sure that the URVE script in Application View is set to Running, or start it manually.
