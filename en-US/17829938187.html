Operator Tasks

This section contains general reference information for operation of the Operator Tasks extension module. For related procedures, see the step-by-step section.

The Operator Tasks application allows you to simply create and monitor building tasks that would normally not be regularly scheduled. For example, suppress alarms in the building for 2 hours during maintenance.

  • An operator task comprises one or multiple commands that will be executed when the task is started.
  • Operator tasks begin to execute when you start them, or at a specified time after that.
  • Operator tasks end after a specified duration, or at a specific date/time.
  • When an operator task ends, a manual or automatic revert can set the commanded parameter back to its original state.

Tasks are created based on pre-programmed templates, so you do not need to know the specific properties and command sequences to configure a task.
Simply select the desired template corresponding to the action you want to perform (for example, Suppress Alarms) and then drag-and-drop target objects of the appropriate type.
