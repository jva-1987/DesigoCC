Setting VMS Access Rights for Video Clip Export

Reference: This task is performed within step 3: Set up User Accounts and Networking of the main video integration workflow.

By default, Desigo CC services run under the local system account. However, to be able to export video clips to a file, Desigo CC services must be set to run under a specific Windows account that has VMS access rights. You can do this in the System Management Console (SMC) as follows:

  1. In the SMC tree, select System.
  2. Open the Settings expander.
  3. The System Account group box shows whether Desigo CC is currently set to run under a Local system account or under a Specific account.
  4. If a Specific account is already set, skip to step 5 below.
  5. If a Local account is currently set, you must change this setting to a Specific account: .
    • Follow the instructions provided in the SMC documentation under System Settings Procedures > Configure the System Account.

  6. NOTICE

    Take great care when changing the system account. Make sure the new specific account meets all the requirements to be the Desigo CC system account, including SQL privileges and rights on other systems, such as those connected via OPC.

    Refer to the SMC documentation under SMC System Settings > System Expander for full details about the system account, and any further actions you need to perform after changing the system account

  7. Once the Desigo CC system account is set to a Specific account you can add the necessary VMS rights to this account:
    • Assign to this specific account the VMS Administrator Role, similarly to how you did for the VideoApiService account. See Assign Administrator Role in the VMS.
